# 📱 Документация Reminder (Напоминания)

## 1. Общая информация
*   **Название:** `Reminder`
*   **Тип:** Активность (Activity)
*   **Назначение:** Этот экран позволяет пользователям устанавливать напоминания о тренировках. Можно выбрать конкретную дату и время или установить таймер (например, «напомни мне через 60 секунд»).
*   **Взаимодействие:** Использует `AlarmManager` для планирования задач и `ReceiverAfterTime` (BroadcastReceiver) для показа уведомлений, когда придет время.

## 2. Переменные (Поля класса)
| Название | Тип | Назначение | Использование |
| :--- | :--- | :--- | :--- |
| `calendar` | `Calendar` | Хранит дату и время, выбранные пользователем. | Передается в `AlarmManager` для точного срабатывания. |
| `etTimeAfter` | `EditText` | Поле для ввода секунд (быстрый таймер). | Значение переводится в миллисекунды и добавляется к текущему времени. |
| `bSetAlarm` | `Button` | Кнопка подтверждения напоминания. | Устанавливает будильник в системе Android. |

## 3. Методы
### Методы: `choiceDate` и `choiceTime`
*   **Что делают:** Открывают стандартные окна выбора даты (**DatePickerDialog**) и времени (**TimePickerDialog**). Это удобные всплывающие окна, которые позволяют выбрать время без ручного ввода цифр.

### Метод кнопки `bAddTimeAlarm`
*   **Что делает:** 
    1. Считывает количество секунд.
    2. Вычисляет время срабатывания (`Текущее время + секунды`).
    3. Создает `PendingIntent`, который указывает на класс `ReceiverAfterTime`.
    4. Регистрирует задачу в системном сервисе `AlarmManager`.

## 6. Взаимодействие с другими компонентами
*   **AlarmManager:** Это системная служба. Даже если пользователь закроет приложение, `AlarmManager` запомнит расписание и «разбудит» код приложения в нужное время.
*   **ReceiverAfterTime:** Когда будильник срабатывает, этот класс получает сигнал и создает уведомление, которое появляется в строке состояния телефона.

## 7. Общая логика
Пользователь устанавливает время. Приложение регистрирует это время в системном будильнике Android. Когда время приходит, система отправляет сигнал (Broadcast) приложению. Приложение ловит этот сигнал и показывает уведомление.

## 8. Объяснение простыми словами
Представьте, что экран `Reminder` — это ваш **личный будильник**. Вы говорите ему, когда хотите заниматься. Приложение оставляет "записку" в календаре системы Android. Когда наступает время, Android "толкает" приложение, и оно кричит: «Эй! Пора учить азбуку Морзе!»
