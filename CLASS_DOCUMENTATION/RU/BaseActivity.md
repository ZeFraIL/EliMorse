# 📱 Документация BaseActivity

## 1. Общая информация
*   **Название:** `BaseActivity`
*   **Тип:** Абстрактный класс (Родительская активность)
*   **Назначение:** Это класс-шаблон. Он содержит код, который используется многими другими экранами приложения. Вместо того чтобы писать один и тот же код для меню или голосового помощника на каждом экране, мы пишем его здесь один раз.
*   **Взаимодействие:** Другие классы, такие как `DashBoard`, `Translate` и `ExerciseGame`, наследуют функционал от этого класса.

## 2. Переменные (Поля класса)
| Название | Тип | Назначение | Использование |
| :--- | :--- | :--- | :--- |
| `user` | `User` | Хранит данные о вошедшем пользователе. | Используется для передачи имени и ID пользователя между экранами. |

## 3. Методы
### Метод: `showPopupMenu`
*   **Тип:** `protected`
*   **Возвращает:** `void` (ничего)
*   **Параметры:**
    *   `anchorView` (`TextView`): Элемент интерфейса, на который нажали для открытия меню.
*   **Что делает:** Создает и показывает всплывающее меню. Обрабатывает нажатия на пункты "Словарь", "О нас" или "Выход".
*   **Вызов:** Когда пользователь нажимает на логотип "Eli-Morse" или кнопку меню.

### Метод: `navigateTo`
*   **Тип:** `protected`
*   **Параметры:**
    *   `activityClass`: Класс экрана, на который нужно перейти.
    *   `sectionName`: Название раздела (для озвучки).
*   **Что делает:** Упрощает переход между экранами. Автоматически передает объект пользователя и озвучивает направление перехода через TTS.

## 8. Объяснение простыми словами
Представьте, что вы строите несколько разных домов из LEGO. Вместо того чтобы собирать новую входную дверь для каждого дома, вы создаете одну **Главную Дверь**. `BaseActivity` — это такая "Главная Дверь" для всех экранов нашего приложения. Она дает всем экранам готовое меню и "голос", чтобы не делать их каждый раз заново.
