# Класс: LogAndReg

## 1. Общая информация
*   **Имя класса:** `LogAndReg`
*   **Тип:** Activity
*   **Назначение:** Этот экран управляет входом пользователя в систему и его регистрацией. Он выступает в роли «контейнера» для двух фрагментов (`LoginFragment` и `RegistryFragment`) и отвечает за логику переключения между ними, а также за связь с базой данных.
*   **Взаимодействие:** Взаимодействует с `LoginFragment` и `RegistryFragment` для получения данных от пользователя, использует `UserRepository` для проверки или сохранения данных в БД и переходит на главный экран (`DashBoard`) при успехе.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `loginFragment` | `LoginFragment` | Объект экрана входа. | Подставляется через `showFragment`. |
| `registryFragment` | `RegistryFragment` | Объект экрана регистрации. | Подставляется через `showFragment`. |
| `rbLogin` | `RadioButton` | Переключатель режима входа. | При нажатии показывает `loginFragment`. |
| `rbRegister` | `RadioButton` | Переключатель режима регистрации. | При нажатии показывает `registryFragment`. |
| `bConfirm` | `Button` | Кнопка действия (Войти / Зарегистрироваться). | Запускает логику проверки/регистрации. |
| `userRepository` | `UserRepository` | Работает с базой данных. | Используется в `handleLogin` и `handleRegistration`. |
| `executorService` | `ExecutorService` | Управление фоновыми потоками. | Позволяет выполнять запросы к БД не в основном потоке интерфейса. |
| `mainThreadHandler`| `Handler` | Управление основным потоком. | Используется для возврата результатов на экран после работы в фоне. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Устанавливает макет экрана.
    2. Вызывает `initElements()` для настройки компонентов.
    3. Настраивает слушателей для RadioButton, чтобы менять фрагменты.
    4. Настраивает кнопку подтверждения для запуска процесса входа или регистрации.
*   **Когда вызывается:** При запуске Activity.

### Имя метода: `showFragment`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `fragment (Fragment)` — фрагмент, который нужно показать.
*   **Логика работы:** Использует `FragmentTransaction`, чтобы заменить текущий фрагмент в контейнере (`R.id.FL`) на новый.
*   **Когда вызывается:** При нажатии на RadioButton или при инициализации.

### Имя метода: `handleRegistration`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Собирает данные (имя, пароль, почта, телефон) из `registryFragment`.
    2. Проверяет поля методом `validateFields()`.
    3. Запускает фоновый поток через `executorService`.
    4. Проверяет в БД, не существует ли уже такой пользователь.
    5. Если нет — сохраняет нового пользователя и переходит в `DashBoard` (через основной поток).
*   **Когда вызывается:** При нажатии на `bConfirm`, если выбран режим «Регистрация».

### Имя метода: `handleLogin`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Получает имя и пароль из `loginFragment`.
    2. Проверяет заполнение полей.
    3. Запускает фоновый поток для поиска пользователя в базе.
    4. Если пользователь найден — переходит в `DashBoard`. Если нет — показывает ошибку.
*   **Когда вызывается:** При нажатии на `bConfirm`, если выбран режим «Вход».

### Имя метода: `validateFields`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `boolean` (true — если все ок, false — если есть пустые поля)
*   **Параметры:** `fields (String...)` — список строк для проверки.
*   **Логика работы:** Проверяет, не является ли какая-либо из строк пустой или равной null.
*   **Когда вызывается:** Перед началом операций с базой данных.

## 4. Жизненный цикл
*   **`onCreate()`**: Инициализирует первое функциональное взаимодействие с приложением. По умолчанию устанавливает фрагмент входа.

## 5. Взаимодействие с интерфейсом (UI)
*   **RadioButton:** Работают как вкладки для переключения между «Входом» и «Регистрацией».
*   **FragmentContainer (`R.id.FL`):** Пустая область в макете, где происходит замена фрагментов.
*   **Button (`bConfirm`):** Выполняет основное действие.

## 6. Взаимодействие с другими компонентами
*   **Фрагменты:** Использует методы-геттеры фрагментов для получения того, что ввел пользователь.
*   **UserRepository:** Скрывает детали работы с SQLite.
*   **DashBoard Activity:** Запускается после успешной проверки пользователя.
*   **ExecutorService:** Критически важен для асинхронной работы. Android запрещает работать с БД в основном потоке, так как это может «заморозить» интерфейс.

## 7. Общая логика класса
`LogAndReg` — это «пропускной пункт». Он дает выбор: «Кто вы?» или «Присоединяйтесь к нам!». Он собирает информацию через маленькие «мини-экраны» (фрагменты). Как только вы нажимаете подтверждение, он отправляется в «картотеку» (базу данных) в фоновом режиме, чтобы проверить данные. Если все в порядке, он пропускает вас в главное меню (`DashBoard`).

## 8. Упрощенное объяснение
Представьте `LogAndReg` как **администратора** на входе в здание. У администратора есть две анкеты: «Вход для своих» и «Регистрация нового гостя». Вы выбираете нужную анкету (RadioButton), заполняете её (фрагменты) и отдаете администратору (кнопка подтверждения). Администратор уходит в архив (базу данных) в подсобное помещение (фоновый поток), чтобы проверить записи. Если всё верно, он выдает вам бейдж (объект пользователя) и пропускает в главный зал (`DashBoard`).

---
**Советы по улучшению:**
*   Безопасность паролей: Пароли хранятся и передаются в открытом виде. В реальном приложении их нужно хешировать.
*   `executorService` никогда не закрывается. Лучше закрывать его в `onDestroy`, чтобы избежать утечек памяти.
