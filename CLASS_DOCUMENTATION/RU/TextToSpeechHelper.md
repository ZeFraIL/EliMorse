# Класс: TextToSpeechHelper

## 1. Общая информация
*   **Имя класса:** `TextToSpeechHelper`
*   **Тип:** Обычный класс (Утилита)
*   **Назначение:** Этот класс предоставляет упрощенный статический способ использования движка Text-to-Speech без необходимости запускать отдельную службу. Он позволяет другим частям кода «заговорить» буквально одной строчкой кода.
*   **Взаимодействие:** Управляет статическим экземпляром системного объекта `TextToSpeech`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `textToSpeech` | `TextToSpeech` | Экземпляр движка озвучки. | Используется в методах `speak()` и `shutdown()`. |

## 3. Методы класса

### Имя метода: `speak`
*   **Тип метода:** `public static`
*   **Логика работы:** Проверяет, инициализирован ли движок, и если да — дает команду немедленно произнести переданный текст.
*   **Когда вызывается:** Когда какой-либо части приложения нужно быстро озвучить сообщение.

### Имя метода: `shutdown`
*   **Тип метода:** `public static`
*   **Логика работы:** Останавливает речь и освобождает системные ресурсы.
*   **Когда вызывается:** При закрытии приложения.

## 4. Жизненный цикл
*   **Неприменимо:** Обычный статический контейнер.

## 5. Общая логика класса
`TextToSpeechHelper` — это **«Мегафон на полке»**. Вам не нужно нанимать целую службу (как `TTS_Service`), чтобы воспользоваться им; вы просто берете его, говорите что нужно и кладете обратно.

## 6. Упрощенное объяснение
Представьте это как **«Быструю кнопку»**. Обычно, чтобы заговорить, нужно много действий (открыть рот, вдохнуть и т.д.). Этот класс позволяет приложению «заговорить», просто нажав одну кнопку.

---
**Советы по улучшению:**
*   **Инициализация:** Код инициализации в данный момент закомментирован. Это означает, что переменная `textToSpeech` всегда будет равна `null`, и метод `speak` фактически ничего не будет делать. Нужно раскомментировать и исправить логику инициализации.
