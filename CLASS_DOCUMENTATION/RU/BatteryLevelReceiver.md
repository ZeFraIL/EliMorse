# Класс: BatteryLevelReceiver

## 1. Общая информация
*   **Имя класса:** `BatteryLevelReceiver`
*   **Тип:** BroadcastReceiver (Приемник широковещательных сообщений)
*   **Назначение:** Этот класс слушает системные сигналы Android об уровне заряда батареи. Его задача — предупредить пользователя, если заряд становится низким (ниже 20%) с помощью всплывающего сообщения, и включить голосовые предупреждения, если заряд упадет ниже 10%.
*   **Взаимодействие:** Регистрируется в системе для получения сообщений `BATTERY_CHANGED`. Использует `My_Toast` для визуальных оповещений и `TTS_Service` для голосовых.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `MAX_REPEATS` | `int` | Лимит голосовых предупреждений (10). | Чтобы не раздражать пользователя бесконечно. |
| `level` | `int` | Текущий процент заряда. | Считывается из системного намерения (Intent). |
| `repeatCount` | `int` | Считает, сколько раз было озвучено предупреждение. | Проверяется в `startVoiceNotifications`. |

## 3. Методы класса

### Имя метода: `onReceive`
*   **Тип метода:** `public`
*   **Логика работы:**
    1. Извлекает целое число «level» из интента.
    2. Если уровень < 20, показывает кастомный тост (сообщение).
    3. Если уровень < 10, показывает диалоговое окно с предупреждением и вызывает `startVoiceNotifications()`.
*   **Когда вызывается:** Автоматически системой Android каждый раз, когда процент заряда батареи меняется.

### Имя метода: `startVoiceNotifications`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Использует `CountDownTimer` для повторения предупреждения каждую минуту (60 000 мс).
    2. При каждом «тике» проверяет, не зарядился ли телефон (level > 20). Если да — отменяет таймер.
    3. Если нет — отправляет Intent в `TTS_Service`, чтобы тот произнес «Низкий уровень заряда».
*   **Когда вызывается:** Когда заряд падает ниже 10%.

## 4. Жизненный цикл
*   **Неприменимо:** Приемник сообщений живет очень недолго — только во время работы метода `onReceive`. Однако запущенный внутри него `CountDownTimer` может продолжать работу (что является не совсем стандартной, но допустимой практикой в данном случае).

## 5. Взаимодействие с другими компонентами
*   **TTS_Service:** Для озвучивания предупреждения.
*   **My_Toast:** Для показа визуального уведомления.
*   **PowerConnectionReceiver:** Вызывает статический метод этого класса для отображения диалога.

## 6. Общая логика класса
`BatteryLevelReceiver` — это **«Часовой безопасности»**. Он зорко следит за индикатором батареи. Как только стрелка заходит в «красную зону», он начинает поднимать шум (визуально и голосом), чтобы пользователь не потерял прогресс в упражнении из-за внезапного выключения телефона.

## 7. Упрощенное объяснение
Представьте этот класс как **предупреждающую сирену**. Когда ваш телефон вот-вот «упадет в обморок» от голода (низкой энергии), включается сирена. Она будет пищать каждую минуту, пока вы не дадите телефону «поесть» (не подключите зарядное устройство).
