# Класс: Welcome

## 1. Общая информация
*   **Имя класса:** `Welcome`
*   **Тип:** Activity (Экран приветствия / Splash Screen)
*   **Назначение:** Это самый первый экран, который видит пользователь при запуске приложения. Его задача — поприветствовать пользователя, показать анимированный логотип приложения и озвучить приветствие через динамик (TTS). Через несколько секунд экран автоматически переключается на страницу входа и регистрации.
*   **Взаимодействие:** Запускает службу `TTS_Service` для воспроизведения приветственного сообщения и открывает экран `LogAndReg`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `ivLogo` | `ImageView` | Отображает логотип приложения. | Используется в `initElements` для запуска анимаций вращения и увеличения. |
| `bStart` | `Button` | Кнопка «Начать» (Start). | Используется в `onCreate` для возможности пропустить таймер и сразу перейти к приложению. |
| `context` | `Context` | Контекст текущего экрана. | Нужен для загрузки анимаций и запуска службы. |
| `cdt` | `CountDownTimer` | Таймер обратного отсчета на 7 секунд. | Используется для автоматического перехода на следующий экран, если пользователь ничего не нажал. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `savedInstanceState (Bundle)` — сохраненное состояние.
*   **Логика работы:**
    1. Вызывает `applyTheme()` для установки правильного стиля оформления.
    2. Устанавливает макет `activity_welcome.xml`.
    3. Инициализирует компоненты через `initElements()`.
    4. Создает и запускает таймер на 7 секунд.
    5. Настраивает кнопку «Начать»: при нажатии таймер отменяется, и вызывается `startMainScreen()`.
*   **Когда вызывается:** Автоматически системой Android при старте приложения.

### Имя метода: `startMainScreen`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Создает `Intent` (намерение) для открытия экрана `LogAndReg`.
    2. Запускает этот экран.
    3. Вызывает `finish()`, чтобы пользователь не мог вернуться на экран приветствия, нажав кнопку «Назад».
*   **Когда вызывается:** Когда таймер заканчивается или когда нажата кнопка «Начать».

### Имя метода: `applyTheme`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Открывает настройки `SharedPreferences` (хранилище настроек) под названием «theme_prefs».
    2. Читает сохраненный режим темы (светлая, темная или системная).
    3. Применяет выбранную тему к приложению.
*   **Когда вызывается:** В самом начале метода `onCreate`.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Связывает переменные Java с элементами XML (`ivLogo`, `bStart`).
    2. Загружает анимацию «увеличения» (zoom in) и запускает ее для логотипа.
    3. Запускает анимацию вращения (на 360 градусов) для логотипа.
    4. Запускает службу `TTS_Service` с приветственным текстом из ресурсов.
*   **Когда вызывается:** Во время работы `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается при создании экрана. Здесь запускается таймер и начинаются анимации.
*   **`onPause()`, `onStop()`**: Стандартное поведение. Примечание: таймер продолжает работать, если его не отменить (отмена происходит только при нажатии на кнопку).

## 5. Взаимодействие с интерфейсом (UI)
*   **ImageView (`ivLogo`):** Показывает логотип. Анимируется с помощью `AnimationUtils`.
*   **Button (`bStart`):** Позволяет войти в приложение сразу. Обрабатывается через `setOnClickListener`.

## 6. Взаимодействие с другими компонентами
*   **Экран LogAndReg:** Открывается через `Intent` после завершения приветствия.
*   **TTS_Service:** Используется для озвучки фразы «Добро пожаловать в Eli Morse» (или аналогичной).
*   **SharedPreferences:** Используется для загрузки предпочтений пользователя по оформлению темы.

## 7. Общая логика класса
Экран `Welcome` — это своего рода «зал ожидания» с представлением. Пока тикают 7 секунд, пользователь видит анимацию и слышит приветствие. Если пользователю некогда ждать, он нажимает «Начать». В обоих случаях это приводит к экрану входа, а сам экран приветствия закрывается.

## 8. Упрощенное объяснение
Представьте экран `Welcome` как **начальные титры** фильма. Нам показывают название (логотип), играет музыка (голос TTS), и создается настроение. Через несколько секунд начинается сам фильм (основное приложение), или вы можете нажать кнопку «Пропустить» (кнопка «Начать»), чтобы сразу перейти к делу.

---
**Советы по улучшению:**
*   Таймер (`cdt`) запускается, но отменяется только при нажатии на кнопку. Если экран будет уничтожен системой (например, при повороте телефона) до окончания таймера, это может вызвать ошибки. Лучше отменять таймер в методе `onDestroy`.
*   Переменная `context` получает значение `this`, что в данном случае избыточно, так как `this` можно использовать напрямую.
