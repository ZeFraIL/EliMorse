# الفئة: Question

## 1. معلومات عامة
*   **اسم الفئة:** `Question`
*   **النوع:** فئة عادية (نموذج بيانات)
*   **الغرض من الفئة:** تمثل هذه الفئة سؤالاً واحداً في التمرين. وهي تحتفظ بـ "المشكلة" (النص الإنجليزي) و "الحل" (نسخة كود مورس).
*   **التفاعلات:** يتم إنشاؤها بواسطة `QuestionRepository` واستهلاكها بواسطة `ExerciseGame` أو `ListeningExerciseActivity`.

## 2. المتغيرات (حقول الفئة)
| الاسم | النوع | الغرض | أين تُستخدم |
| :--- | :--- | :--- | :--- |
| `questionText` | `String` | الكلمة/الحرف/الرقم المراد ترجمته. | يُعرض للمستخدم. |
| `morseAnswer` | `String` | كود مورس الصحيح للسؤال. | يتم مقارنته بمدخلات المستخدم. |

## 3. أساليب الفئة (Methods)

### اسم الأسلوب: `Question` (الباني - Constructor)
*   **النوع:** `public`
*   **المنطق:** يقوم بتخزين النص وإجابته.
*   **متى يتم استدعاؤه:** داخل `QuestionRepository` عند إنشاء أسئلة جديدة.

### أسماء الأساليب: `getQuestionText` / `getMorseAnswer`
*   **النوع:** `public`
*   **المنطق:** محصلات (getters) قياسية.

## 4. دورة الحياة (Lifecycle)
*   **لا ينطبق:** كائن بيانات بسيط.

## 5. المنطق العام للفئة
تعتبر `Question` بمثابة **بطاقة استذكار**. لها وجه أمامي (السؤال) ووجه خلفي (الإجابة). تحتفظ بالإجابة الصحيحة مخفية حتى يحتاج التطبيق للتحقق منها.

## 6. شرح بكلمات بسيطة
فكر في `Question` كأنها **أحجية**. تسأل الأحجية: "كيف تقول HELLO بلغة مورس؟". تحتفظ هذه الفئة بكل من السؤال "HELLO" والإجابة السرية ".... . .-.. .-.. ---".
