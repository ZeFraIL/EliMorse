# Класс: MorseTranslator

## 1. Общая информация
*   **Имя класса:** `MorseTranslator`
*   **Тип:** Обычный класс (Служебный / Логика)
*   **Назначение:** Это «мозг» функций перевода в приложении. Класс знает весь алфавит азбуки Морзе и предоставляет методы для преобразования английских предложений в точки и тире и обратно.
*   **Взаимодействие:** Используется активностью `Translate` и репозиторием `QuestionRepository`. Напрямую с интерфейсом (UI) не взаимодействует.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `morseMap` | `Map<String, String>` | Словарь для перевода Текст -> Морзе. | Используется в `toMorse` для быстрого поиска. |
| `textMap` | `Map<String, String>` | Словарь для перевода Морзе -> Текст. | Используется в `toText` для быстрого поиска. |

## 3. Методы класса

### Имя метода: `toMorse`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `String` (версия на языке Морзе)
*   **Параметры:** `text (String)` — английский текст.
*   **Логика работы:**
    1. Переводит текст в верхний регистр.
    2. Проходит циклом по каждой букве.
    3. Ищет букву в `morseMap`.
    4. Добавляет точки/тире в `StringBuilder`.
    5. Ставит один пробел между буквами и символ `/` между словами.
*   **Когда вызывается:** Когда пользователь переводит текст или когда генерируется вопрос для упражнения.

### Имя метода: `toText`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `String` (английская версия)
*   **Параметры:** `morseCode (String)` — строка из точек и тире.
*   **Логика работы:**
    1. Разделяет строку по символу `/`, чтобы получить отдельные слова.
    2. Разделяет каждое слово по пробелам, чтобы получить отдельные буквы Морзе.
    3. Ищет символ в `textMap`.
    4. Если символ не найден, добавляет знак `?`.
*   **Когда вызывается:** Когда пользователь переводит код Морзе обратно в текст.

### Имя метода: `initMorseMap`
*   **Тип метода:** `private`
*   **Логика работы:** Вручную заполняет словари в соответствии с международным стандартом азбуки Морзе (A=.-, B=-... и т.д.).
*   **Когда вызывается:** Автоматически в конструкторе при создании объекта.

## 4. Жизненный цикл
*   **Неприменимо:** Существует в памяти как вспомогательный объект.

## 5. Взаимодействие с интерфейсом (UI)
*   **Отсутствует:** Этот класс содержит чистую логику.

## 6. Взаимодействие с другими компонентами
*   **Активность Translate:** Полагается на этот класс для выполнения основной задачи экрана.

## 7. Общая логика класса
`MorseTranslator` работает как **«Автоматизированный словарь»**. Он берет на себя все правила форматирования (например, добавление пробелов между буквами), чтобы другим частям приложения не приходилось беспокоиться о деталях грамматики азбуки Морзе.

## 8. Упрощенное объяснение
Представьте `MorseTranslator` как **секретное дешифровальное кольцо**. Вы даете ему слово, и оно превращает его в код. Вы даете ему код, и оно превращает его обратно в слово. Это единственная часть приложения, которая на самом деле «умеет» говорить на языке Морзе.

---
**Советы по улучшению:**
*   **Эффективность:** Метод `toText` использует `replaceAll("\\s+", " ")`, что может работать медленно на очень длинных строках, но для обычных сообщений Морзе это вполне допустимо.
*   **Спецсимволы:** В данный момент поддерживаются только A-Z и 0-9. Добавление знаков препинания (точки, запятые) сделало бы переводчик более функциональным.
