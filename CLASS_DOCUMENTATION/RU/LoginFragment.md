# Класс: LoginFragment

## 1. Общая информация
*   **Имя класса:** `LoginFragment`
*   **Тип:** Fragment (Фрагмент)
*   **Назначение:** Этот класс представляет собой часть экрана `LogAndReg`. Он отвечает за интерфейс входа в систему, предоставляя поля для ввода имени пользователя и пароля.
*   **Взаимодействие:** Размещается внутри `LogAndReg`. Активность вызывает методы этого фрагмента, чтобы получить текст, введенный пользователем.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `etNameL` | `EditText` | Поле ввода для имени пользователя. | Инициализируется в `onCreateView` и считывается в `getName`. |
| `etPasswordL`| `EditText` | Поле ввода для пароля. | Инициализируется в `onCreateView` и считывается в `getPassword`. |

## 3. Методы класса

### Имя метода: `onCreateView`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `View` (созданный интерфейс)
*   **Логика работы:**
    1. Загружает макет `fragment_login.xml`.
    2. Находит элементы `EditText` внутри этого макета.
    3. Возвращает готовое визуальное представление системе для отображения.
*   **Когда вызывается:** Когда фрагмент создается и добавляется на экран.

### Имена методов: `getName` / `getPassword`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `String` (строка текста)
*   **Логика работы:** Просто возвращает текущий текст, находящийся в соответствующих полях `EditText`.
*   **Когда вызывается:** Родительской активностью (`LogAndReg`), когда пользователь нажимает кнопку «Подтвердить».

## 4. Жизненный цикл
*   **`onCreateView()`**: Самый важный метод жизненного цикла для фрагментов. Именно здесь строится интерфейс и привязывается к коду Java.

## 5. Взаимодействие с интерфейсом (UI)
*   **EditText:** Используются для ввода обычного и скрытого (пароль) текста.

## 6. Взаимодействие с другими компонентами
*   **LogAndReg (Activity):** Выступает в роли родителя. Фрагмент сам не выполняет сложной логики (например, проверку в базе данных); он просто хранит данные для активности.

## 7. Общая логика класса
`LoginFragment` — это **«Сборщик данных»**. Он предоставляет текстовые поля и ждет, пока пользователь что-то напечатает. Он не знает, *зачем* он собирает данные и *куда* они пойдут; он просто выдает имя и пароль всякий раз, когда активность об этом просит.

## 8. Упрощенное объяснение
Представьте `LoginFragment` как **небольшой бумажный бланк** внутри большой папки. Папка (`LogAndReg`) показывает этот бланк пользователю. Вы вписываете свое имя и пароль в бланк. Когда вы закончили, папка «считывает» то, что вы написали на бумаге.
