# Класс: GuideList

## 1. Общая информация
*   **Имя класса:** `GuideList`
*   **Тип:** Activity
*   **Назначение:** Этот экран выполняет роль полного словаря азбуки Морзе. Он считывает список всех символов и соответствующих им букв/цифр из текстового файла в ресурсах приложения и отображает их в виде прокручиваемого списка.
*   **Взаимодействие:** Наследует `BaseActivity`. Использует модель данных `MorseCode` и читает данные из файла `R.raw.all_morse`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `info` | `ArrayList<String>`| Хранит строки для отображения в списке. | Заполняется при чтении файла и передается адаптеру. |
| `lv` | `ListView` | Элемент списка на экране. | Отображает пункты словаря. |
| `user` (наследуется) | `User` | Данные текущего пользователя. | Используется для корректной работы меню. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Логика работы:**
    1. Устанавливает макет `activity_guide_list.xml`.
    2. Вызывает `initElements()` для загрузки данных и настройки интерфейса.
    3. Настраивает всплывающее меню.
*   **Когда вызывается:** При нажатии на «Словарь Морзе» в меню справочника.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Получает объект пользователя.
    2. Открывает файл `all_morse` из папки `res/raw`.
    3. Использует `BufferedReader` для построчного чтения.
    4. Поскольку файл структурирован так, что на одну запись приходится две строки (код, затем буква), метод считывает по две строки за раз.
    5. Создает объект `MorseCode` для каждой пары и добавляет его текстовое представление в список `info`.
    6. Обрабатывает возможные ошибки чтения (IOException), выводя сообщение в лог и на экран.
    7. Создает `ArrayAdapter` и привязывает его к `ListView`.
*   **Когда вызывается:** Внутри метода `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Инициирует загрузку и отображение данных.

## 5. Взаимодействие с интерфейсом (UI)
*   **ListView (`lv`):** Основной способ взаимодействия пользователя со словарем через прокрутку.

## 6. Взаимодействие с другими компонентами
*   **Класс MorseCode:** Используется для форматирования данных в удобный вид «Код -> Буква».
*   **Сырые ресурсы (Raw):** Читает данные из локального текстового файла, упакованного в приложение.

## 7. Общая логика класса
Класс `GuideList` — это **«Загрузчик данных»**. Он превращает статический текстовый файл в динамичный, прокручиваемый список на экране телефона. Он гарантирует, что информация представлена в чистом и единообразном формате.

## 8. Упрощенное объяснение
Представьте `GuideList` как **цифровую шпаргалку**. В памяти приложения есть «скрытый» список всех точек и тире. Когда вы открываете этот экран, приложение достает этот список и аккуратно распечатывает его на вашем экране, чтобы вы могли подсмотреть любой символ, который забыли.

---
**Советы по улучшению:**
*   **Форматирование:** Сейчас метод `MorseCode.toString()` использует пробелы для выравнивания. На некоторых экранах колонки могут «поехать». Использование собственного макета для элементов списка выглядело бы профессиональнее.
*   **Чтение файлов:** Использование блока `try-with-resources` — это хорошая практика, так как файл закрывается автоматически даже при возникновении ошибки.
