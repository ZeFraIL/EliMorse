# Класс: Question

## 1. Общая информация
*   **Имя класса:** `Question`
*   **Тип:** Обычный класс (Модель данных)
*   **Назначение:** Этот класс представляет один вопрос в упражнении. Он хранит саму «задачу» (английский текст) и её «решение» (код Морзе).
*   **Взаимодействие:** Генерируется репозиторием `QuestionRepository` и используется активностями `ExerciseGame` или `ListeningExerciseActivity`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `questionText` | `String` | Слово, буква или число для перевода. | Показывается пользователю на экране. |
| `morseAnswer` | `String` | Правильный код Морзе для этого вопроса. | Сравнивается с тем, что ввел пользователь. |

## 3. Методы класса

### Имя метода: `Question` (Конструктор)
*   **Тип метода:** `public`
*   **Логика работы:** Сохраняет текст вопроса и правильный ответ.
*   **Когда вызывается:** Внутри `QuestionRepository` при создании новых вопросов.

### Имена методов: `getQuestionText` / `getMorseAnswer`
*   **Тип метода:** `public`
*   **Логика работы:** Стандартные методы для получения данных (геттеры).

## 4. Жизненный цикл
*   **Неприменимо:** Обычный объект данных.

## 5. Общая логика класса
`Question` — это **«Карточка для викторины»**. На одной её стороне написан вопрос, а на другой — правильный ответ. Она хранит ответ скрытым до тех пор, пока приложению не понадобится проверить знания пользователя.

## 6. Упрощенное объяснение
Представьте `Question` как **загадку**. Загадка звучит так: «Как сказать ПРИВЕТ на языке Морзе?». Этот класс хранит и саму загадку «ПРИВЕТ», и секретный ответ «.--. .-. .. .-- . -».
