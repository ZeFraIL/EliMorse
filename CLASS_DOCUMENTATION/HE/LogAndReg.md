# מחלקה: LogAndReg

## 1. מידע כללי
*   **שם המחלקה:** `LogAndReg`
*   **סוג:** Activity
*   **מטרת המחלקה:** מסך זה מנהל הן את ההתחברות (Login) והן את ההרשמה (Registration) של המשתמש. הוא פועל כ"מיכל" עבור שני פרגמנטים (`LoginFragment` ו-`RegistryFragment`) ומנהל את הלוגיקה של המעבר ביניהם ואת התקשורת מול בסיס הנתונים.
*   **אינטראקציה:** המחלקה מתקשרת עם הפרגמנטים כדי לקבל קלט מהמשתמש, משתמשת ב-`UserRepository` כדי לבדוק או לשמור נתונים בבסיס הנתונים, ועוברת למסך ה-`DashBoard` במידה והתהליך הצליח.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `loginFragment` | `LoginFragment` | אובייקט של מסך ההתחברות. | מוצג דרך המתודה `showFragment`. |
| `registryFragment` | `RegistryFragment` | אובייקט של מסך ההרשמה. | מוצג דרך המתודה `showFragment`. |
| `rbLogin` | `RadioButton` | כפתור בחירה למצב התחברות. | מאזין ללחיצה כדי להציג את ה-`loginFragment`. |
| `rbRegister` | `RadioButton` | כפתור בחירה למצב הרשמה. | מאזיн ללחיצה כדי להציג את ה-`registryFragment`. |
| `bConfirm` | `Button` | כפתור האישור (התחברות / הרשמה). | מפעיל את לוגיקת האימות. |
| `userRepository` | `UserRepository` | מטפל בשאילתות לבסיס הנתונים. | בשימוש ב-`handleLogin` ו-`handleRegistration`. |
| `executorService` | `ExecutorService` | מנהל תהליכונים ברקע. | משמש להרצת שאילתות בסיס נתונים מחוץ לתהליכון הראשי. |
| `mainThreadHandler`| `Handler` | מנהל התהליכון הראשי (UI). | משמש להחזרת תוצאות לממשק המשתמש לאחר עבודה ברקע. |

## 3. מתודות (שיטות) המחלקה

### שם המתודה: `onCreate`
*   **סוג המתודה:** `protected`
*   **ערך חזור:** `void`
*   **לוגיקה:**
    1. מגדירה את עיצוב הממשק.
    2. קוראת ל-`initElements()` להגדרת הרכיבים.
    3. מגדירה מאזינים ל-RadioButtons כדי להחליף בין הפרגמנטים.
    4. מגדירה מאזין לכפתור האישור להתחלת תהליך ההתחברות או ההרשמה.
*   **מתי נקראת:** כשה-Activity מתחיל.

### שם המתודה: `showFragment`
*   **סוג המתודה:** `private`
*   **ערך חזור:** `void`
*   **פרמטרים:** `fragment (Fragment)` — הפרגמנט שיוצג.
*   **לוגיקה:** משתמשת ב-`FragmentTransaction` כדי להחליף את הפרגמנט הנוכחי בתוך המיכל (`R.id.FL`) בפרגמנט החדש.
*   **מתי נקראת:** כשלוחצים על RadioButton או בזמן האתחול.

### שם המתודה: `handleRegistration`
*   **סוג המתודה:** `private`
*   **ערך חזור:** `void`
*   **לוגיקה:**
    1. אוספת נתונים (שם, סיסמה, אימייל, טלפון) מה-`registryFragment`.
    2. מאמתת שהשדות אינם ריקים בעזרת `validateFields()`.
    3. מפעילה תהליכון רקע דרך `executorService`.
    4. בודקת אם המשתמש כבר קיים בבסיס הנתונים.
    5. אם לא, שומרת את המשתמש החדש ועוברת ל-`DashBoard` דרך התהליכון הראשי.
*   **מתי נקראת:** כשלוחצים על `bConfirm` כשמצב "הרשמה" נבחר.

### שם המתודה: `handleLogin`
*   **סוג המתודה:** `private`
*   **ערך חזור:** `void`
*   **לוגיקה:**
    1. אוספת שם וסיסמה מה-`loginFragment`.
    2. מאמתת שהשדות אינם ריקים.
    3. מפעילה תהליכון רקע למציאת המשתמש בבסיס הנתונים.
    4. אם נמצא — עוברת ל-`DashBoard`. אם לא — מציגה הודעת שגיאה.
*   **מתי נקראת:** כשלוחצים על `bConfirm` כשמצב "התחברות" נבחר.

### שם המתודה: `validateFields`
*   **סוג המתודה:** `private`
*   **ערך חזור:** `boolean` (true אם תקין, false אחרת)
*   **פרמטרים:** `fields (String...)` — מספר משתנה של מחרוזות לבדיקה.
*   **לוגיקה:** בודקת אם אחת המחרוזות שהועברו היא null או ריקה.
*   **מתי נקראת:** לפני תחילת פעולות בבסיס הנתונים.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`**: מאתחלת את האינטראקציה הפונקציונלית הראשונה של האפליקציה. מגדירה את הפרגמנט הדיפולטיבי כ-`loginFragment`.

## 5. אינטראקציה עם ממשק המשתמש (UI)
*   **RadioButtons:** משמשים כ"לשוניות" למעבר בין התחברות להרשמה.
*   **FragmentContainer (`R.id.FL`):** מקום ייעודי בעיצוב שבו הפרגמנטים מוחלפים.
*   **Button (`bConfirm`):** מבצע את הפעולה העיקרית.

## 6. אינטראקציה עם רכיבים אחרים
*   **פרגמנטים:** משתמשת במתודות "גטר" (getters) בתוך הפרגמנטים כדי לקבל את קלט המשתמש.
*   **UserRepository:** מחלקה המפשטת את פעולות ה-SQLite.
*   **מסך DashBoard:** נפתח לאחר אימות מוצלח.
*   **ExecutorService:** חיוני לפעולות אסינכרוניות. אנדרואיד לא מאפשרת עבודה מול בסיס נתונים בתהליכון הראשי כדי לא להקפיא את הממשק.

## 7. לוגיקה כללית של המחלקה
`LogAndReg` היא שער הכניסה. היא מציעה בחירה: "מי אתה?" או "הצטרף אלינו!". היא אוספת מידע דרך "מיני-מסכים" (פרגמנטים). ברגע שמאשרים, היא הולכת ל"ארכיון" (בסיס הנתונים) ברקע כדי לבדוק את הפרטים. אם הכל תקין, היא מאפשרת כניסה ללוח הבקרה (`DashBoard`).

## 8. הסבר במילים פשוטות
חשבו על `LogAndReg` כעל **פקיד קבלה** בכניסה לבניין. לפקיד יש שני סוגי טפסים: טופס "כניסה למנויים" וטופס "הרשמה למנוי חדש". אתם בוחרים את הטופס הרצוי (RadioButtons), ממלאים אותו (הפרגמנטים) ומגישים לפקיד (כפתור האישור). הפקיד ניגש לבדוק ברישומים (בסיס הנתונים) במשרד האחורי (תהליכון הרקע). אם הפרטים נכונים, הוא נותן לכם תג זיהוי (אובייקט המשתמש) ומכניס אתכם לאולם הראשי (`DashBoard`).

---
**באגים/שיפורים:**
*   אבטחת סיסמאות: הסיסמאות נשמרות ומועברות כטקסט רגיל. באפליקציה אמיתית יש להשתמש בהצפנה (Hashing).
*   ה-`executorService` לא נסגר אף פעם. כדאי לסגור אותו ב-`onDestroy` כדי למנוע דליפות זיכרון.
