# מחלקה: ExerciseGame

## 1. מידע כללי
*   **שם המחלקה:** `ExerciseGame`
*   **סוג:** Activity
*   **מטרת המחלקה:** ה-Activity הזה מספק את סביבת התרגול האינטראקטיבית לקידוד טקסט לקוד מורס. למשתמש מוצגת מילה, אות או מספר והוא חייב להזין את הנקודות והקווים הנכונים. המחלקה עוקבת אחר טעויות ושומרת את הביצועים הסופיים לבסיס הנתונים.
*   **אינטראקציה:** המחלקה יורשת מ-`BaseActivity`. היא משתמשת ב-`QuestionRepository` כדי לקבל שאלות אקראיות, ב-`HelperDB` כדי לשמור תוצאות, וחוזרת ל-`DashBoard` בסיום.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `questionNum` | `int` | אינדקס השאלה הנוכחית (0 עד 9). | משמש למעקב אחר ההתקדמות ברשימת השאלות. |

## 3. מתודות (שיטות) המחלקה

### שם המתודה: `onCreate`
*   **סוג המתודה:** `protected`
*   **לוגיקה:**
    1. מגדירה את העיצוב.
    2. קוראת ל-`initElements()` להכנת השאלות והממשק.
    3. מגדירה מאזינים לכפתורי נקודה (`.`), קו (`-`) ורווח (`/`).
    4. מגדירה מאזין לכפתור "Ready" לבדיקת התשובה.
    5. מגדירה מאזינים לכפתורי "Clear" ו-"Remove".
*   **מתי נקראת:** כשה-Activity מתחיל.

### שם המתודה: `checkAnswer`
*   **סוג המתודה:** `private`
*   **לוגיקה:**
    1. משווה בין `stAnswer` לבין `currentQuestion.getMorseAnswer()`.
    2. **אם נכון:** עוברת לשאלה הבאה. אם זו הייתה השאלה האחרונה, היא שומרת תוצאות ומציגה דיאלוג סיום.
    3. **אם שגוי:** מעלה את `failedAttempts` ואת `nOfTry`. מעדכנת את נקודות המחוון (O/Ø). לאחר 3 ניסיונות כושלים, היא מציגה דיאלוג "נכשל".
*   **מתי נקראת:** כשהמשתמש לוחץ על "Ready".

### שם המתודה: `appendToAnswer`
*   **סוג המתודה:** `private`
*   **פרמטרים:** `s (String)` — התו שברצוננו להוסיף.
*   **לוגיקה:** מוסיפה את הסימן ל-`stAnswer` ומעדכנת את התצוגה, מוגבל ל-30 תווים.
*   **מתי נקראת:** כשלוחצים על כפתורי נקודה, קו או רווח.

### שם המתודה: `saveResults`
*   **סוג המתודה:** `private`
*   **לוגיקה:** משתמשת בתהליכון רקע כדי להכניס את סוג התרגול, מספר הטעויות, התאריך והמשתמש לבסיס הנתונים בעזרת `ContentValues`.
*   **מתי נקראת:** לאחר סיום מוצלח של כל 10 השאלות.

### שם המתודה: `initElements`
*   **סוג המתודה:** `private`
*   **לוגיקה:**
    1. מקבלת משתמש וסוג משחק מה-Intent.
    2. משתמשת ב-`QuestionRepository` כדי ליצור 10 שאלות אקראיות.
    3. מקשרת את כל רכיבי הממשק.
    4. קוראת ל-`resetAfterQuestion()` כדי להתחיל את השאלה הראשונה.
*   **מתי נקראת:** במהלך ה-`onCreate`.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`**: מגדירה את כל סשן המשחק.

## 5. אינטראקציה עם ממשק המשתמש (UI)
*   **כפתורי קלט:** נקודה, קו, רווח.
*   **כפתורי בקרה:** Ready (אישור), Clear (מחיקת הכל), Remove One (מחיקת תו אחרון).
*   **מחוונים:** `tvTryNum1`, `tvTryNum2`, `tvTryNum3` משנים צבע או טקסט כדי להראות ניסיונות שנוצלו.

## 6. אינטראקציה עם רכיבים אחרים
*   **QuestionRepository:** חיוני לקבלת תוכן מגוון.
*   **HelperDB:** מתעד את התקדמות המשתמש עבור מסך ההיסטוריה.
*   **AlertDialog:** משמש להודעות "סוף משחק" ו-"נכשל".

## 7. לוגיקה כללית של המחלקה
`ExerciseGame` הוא **המנוע הראשי** לתרגול. הוא עובד בלולאה: הצג שאלה -> אסוף קלט -> בדוק תשובה -> שאלה הבאה. הוא שומר תיעוד של ההצלחה שלכם ומוודא שהנתונים נשמרים לצמיתות לפני שאתם עוזבים את המסך.

## 8. הסבר במילים פשוטות
חשבו על `ExerciseGame` כעל **מנחה חידון**. המנחה מראה לכם כרטיס עם מילה. יש לכם מקלדת קטנה עם שלושה סמלים בלבד. אתם מקלידים את הסמלים שאתם חושבים שהם נכונים. המנחה עוקב אחר הנקודות והטעויות שלכם. בסוף, המנחה כותב את הציון שלכם בספר גדול (בסיס הנתונים) כדי שתוכלו להסתכל בו מאוחר יותר.

---
**באגים/שיפורים:**
*   לוגיקת `updateTryIndicators` הייתה חסרה בקטע הקוד שסופק (הוחלפה בהערה). היא אמורה לשנות את הצבע של סמלי ה-"O" לאדום כשניסיון נכשל.
*   יש לסגור את ה-`executorService` בצורה מסודרת.
