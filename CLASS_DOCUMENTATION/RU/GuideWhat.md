# Класс: GuideWhat

## 1. Общая информация
*   **Имя класса:** `GuideWhat`
*   **Тип:** Activity
*   **Назначение:** Этот экран объясняет историю и основные концепции азбуки Морзе. Это текстовая информационная страница.
*   **Взаимодействие:** Наследует `BaseActivity`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `user` (наследуется) | `User` | Данные текущего пользователя. | Используется для корректной работы меню. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Логика работы:**
    1. Устанавливает макет `activity_guide_what.xml`.
    2. Инициализирует элементы.
    3. Настраивает всплывающее меню.
*   **Когда вызывается:** Когда пользователь выбирает пункт «Что такое азбука Морзе?» в справочнике.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Логика работы:** Извлекает объект пользователя из Intent.
*   **Когда вызывается:** Во время `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Отвечает за вывод информации на экран при его создании.

## 5. Взаимодействие с интерфейсом (UI)
*   **TextViews:** Используются в макете XML для отображения исторического текста и определений.

## 6. Взаимодействие с другими компонентами
*   **Отсутствует:** Это статичный информационный экран.

## 7. Общая логика класса
`GuideWhat` — это **«Страница учебника»**. Её единственная задача — оставаться неподвижной и позволить пользователю прочитать интересные факты о том, как Самуил Морзе изобрел свой код.

## 8. Упрощенное объяснение
Представьте `GuideWhat` как **статью в энциклопедии**. Вы открываете её, чтобы узнать ответы на вопросы «Кто?», «Когда?» и «Зачем?».

---
**Советы по улучшению:**
*   Класс предельно прост и хорошо подходит для отображения статического текста.
*   Если текста станет слишком много, стоит добавить `ScrollView` в макет XML.
