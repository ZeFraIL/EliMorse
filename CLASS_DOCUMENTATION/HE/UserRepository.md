# 🧩 תיעוד UserRepository

## 1. מידע כללי
*   **שם:** `UserRepository`
*   **סוג:** מחלקת Java רגילה (Repository / Data Access Object)
*   **מטרה:** מחלקה זו משמשת כגשר בין הלוגיקה של האפליקציה לבין מסד הנתונים בכל הנוגע למשתמשים. היא מטפלת במשימות כמו מציאת משתמש לפי שם וסיסמה או שמירת משתמש חדש.
*   **אינטראקציה:** בשימוש על ידי `LogAndReg` ו-`DashBoard`. היא מתקשרת ישירות עם `HelperDB`.

## 2. משתנים
| שם | סוג | מטרה |
| :--- | :--- | :--- |
| `helperDB` | `HelperDB` | מנהל את החיבור לקובץ ה-SQLite. |

## 3. שיטות (Methods)
### שיטה: `findUser`
*   **מה היא עושה:** 
    1. פותחת את מסד הנתונים לקריאה.
    2. מריצה שאילתה המחפשת שורה שבה השם והסיסמה תואמים.
    3. אם נמצאה התאמה, היא הופכת את השורה מהמסד לאובייקט מסוג `User`.
    4. מחזירה את האובייקט.

### שיטה: `registerUser`
*   **מה היא עושה:** 
    1. פותחת את מסד הנתונים לכתיבה.
    2. מכינה את הנתונים (שם, סיסמה, אימייל, טלפון).
    3. מכניסה אותם לטבלת ה-`Users`.
    4. מחזירה את ה-ID הייחודי של המשתמש החדש.

## 7. לוגיקה כללית
המחלקה הזו מרכזת את כל ה"שאלות" שיש לנו לגבי משתמשים מול מסד הנתונים. במקום לכתוב קוד SQL בכל מקום באפליקציה, אנחנו כותבים אותו כאן פעם אחת. זה שומר על הקוד נקי ומסודר.

## 8. הסבר במילים פשוטות
חשבו על `UserRepository` כעל **ספרן**. כשאתם רוצים ספר מסוים (משתמש), אתם לא הולכים לחפש במדפים בעצמכם. אתם שואלים את הספרן: "תוכל למצוא את המשתמש בשם אליה עם הסיסמה הזו?". הספרן הולך למחסן (מסד הנתונים), מוצא את המידע ומביא לכם אותו.
