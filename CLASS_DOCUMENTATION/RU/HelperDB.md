# 🧩 Документация HelperDB

## 1. Общая информация
*   **Название:** `HelperDB`
*   **Тип:** Обычный Java-класс (Наследует `SQLiteOpenHelper`)
*   **Назначение:** Этот класс отвечает за создание и управление локальным файлом базы данных под названием `Info.db`. Он определяет структуру (таблицы и столбцы) того, где хранятся все данные пользователей и результаты упражнений.
*   **Взаимодействие:** Используется классом `UserRepository` и другими активностями, которым нужно сохранять результаты.

## 2. Переменные (Константы)
| Название | Тип | Назначение |
| :--- | :--- | :--- |
| `DATABASE_NAME` | `String` | Имя файла на телефоне: "Info.db". |
| `TABLE_USER` | `String` | Название таблицы пользователей: "Users". |
| `COLUMN_USER_NAME` | `String` | Название столбца для хранения имени пользователя. |
| `TABLE_EXERCISE` | `String` | Название таблицы результатов: "Exercises". |

## 3. Методы
### Метод: `onCreate`
*   **Тип:** `public`
*   **Что делает:** Этот метод запускается **только один раз** при первой установке приложения, когда база данных создается впервые. Он выполняет SQL-команды для создания таблиц `Users` и `Exercises`.

### Метод: `onUpgrade`
*   **Что делает:** Запускается, если вы измените версию базы данных (`DATABASE_VERSION`). В этом приложении он удаляет старые таблицы и создает новые.

## 5. Структура базы данных
1.  **Таблица Users**: Хранит ID, имя, пароль, почту и телефон.
2.  **Таблица Exercises**: Хранит ID, тип игры, количество ошибок, дату и имя пользователя.

## 7. Общая логика
Этот класс выступает в роли "Архитектора" базы данных. Обычно он не ищет данные; он просто следит за тем, чтобы "здание" (структура базы данных) было готово и имело правильные "комнаты" (таблицы) для данных.

## 8. Объяснение простыми словами
Представьте, что `HelperDB` — это **чертеж** шкафа для документов. Он решает, сколько в шкафу будет ящиков (Таблицы) и какие наклейки будут на каждой папке (Столбцы). Он гарантирует, что при первом запуске шкаф будет собран правильно, чтобы в него можно было складывать информацию.
