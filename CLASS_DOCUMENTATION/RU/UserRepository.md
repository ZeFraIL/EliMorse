# 🧩 Документация UserRepository

## 1. Общая информация
*   **Название:** `UserRepository`
*   **Тип:** Обычный Java-класс (Репозиторий / DAO)
*   **Назначение:** Этот класс служит мостом между логикой приложения и базой данных во всем, что касается пользователей. Он выполняет такие задачи, как поиск пользователя по имени и паролю или сохранение нового пользователя.
*   **Взаимодействие:** Используется в `LogAndReg` и `DashBoard`. Напрямую обращается к `HelperDB`.

## 2. Переменные (Поля класса)
| Название | Тип | Назначение |
| :--- | :--- | :--- |
| `helperDB` | `HelperDB` | Помощник базы данных, который управляет подключением к файлу SQLite. |

## 3. Методы
### Метод: `findUser`
*   **Тип:** `public`
*   **Возвращает:** `User` (Объект найденного пользователя или `null`, если он не найден)
*   **Что делает:** 
    1. Открывает базу данных для чтения.
    2. Выполняет запрос на поиск строки, где имя И пароль совпадают.
    3. Если совпадение найдено, превращает строку из БД в объект класса `User`.
    4. Закрывает соединение и возвращает объект.

### Метод: `registerUser`
*   **Тип:** `public`
*   **Что делает:** Собирает данные пользователя (имя, пароль, почта, телефон) и вставляет их в таблицу `Users`. Возвращает уникальный ID новой записи.

## 7. Общая логика
Этот класс собирает в одном месте все "вопросы", которые мы можем задать базе данных о пользователях. Вместо того чтобы писать SQL-код во всем приложении, мы пишем его здесь один раз. Это делает код чище.

## 8. Объяснение простыми словами
Представьте, что `UserRepository` — это **библиотекарь**. Когда вам нужна конкретная книга (Пользователь), вы не бродите сами по полкам. Вы спрашиваете библиотекаря: "Найдите пользователя Элия с таким паролем". Библиотекарь идет в хранилище (База данных), находит информацию и приносит её вам.
