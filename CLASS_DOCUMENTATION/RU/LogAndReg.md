# 📱 Документация LogAndReg (Активность входа и регистрации)

## 1. Общая информация
*   **Название:** `LogAndReg`
*   **Тип:** Активность (Activity)
*   **Назначение:** Эта активность отвечает за вход пользователя в приложение. Она предоставляет две возможности: вход для существующих пользователей или регистрация для новых. Она управляет переключением между экранами входа и регистрации (фрагментами).
*   **Взаимодействие:** Взаимодействует с `LoginFragment` и `RegistryFragment` для получения ввода от пользователя и использует `UserRepository` для сохранения или проверки данных в базе.

## 2. Переменные (Поля класса)
| Название | Тип | Назначение | Использование |
| :--- | :--- | :--- | :--- |
| `loginFragment` | `LoginFragment` | Компонент экрана входа. | Отображается, когда выбран пункт "Вход". |
| `registryFragment` | `RegistryFragment` | Компонент экрана регистрации. | Отображается, когда выбран пункт "Регистрация". |
| `userRepository` | `UserRepository` | Инструмент для работы с БД. | Используется для поиска пользователей или их регистрации. |
| `executorService` | `ExecutorService` | Менеджер фоновых потоков. | Позволяет работать с базой данных, не замораживая экран. |

## 3. Методы
### Метод: `showFragment`
*   **Тип:** `private`
*   **Параметры:** `fragment` (`Fragment`) — экран для отображения.
*   **Что делает:** Заменяет текущее содержимое в контейнере `FrameLayout` на выбранный фрагмент (вход или регистрация).

### Метод: `handleRegistration`
*   **Что делает:** Собирает данные из фрагмента регистрации, проверяет их и в фоновом потоке выясняет, существует ли уже такой пользователь. Если нет — создает нового.

### Метод: `handleLogin`
*   **Что делает:** Получает имя и пароль из фрагмента входа и ищет пользователя в базе в фоновом потоке. Если пользователь найден — переходит на главный экран (`DashBoard`).

## 8. Объяснение простыми словами
Представьте **стойку администратора (Reception)** в клубе. Администратор спрашивает: "Вы уже член клуба или хотите вступить?". В зависимости от ответа, он дает вам нужную анкету (Фрагмент). Когда вы её заполняете, он проверяет в компьютере (База данных), можно ли вас пропустить.
