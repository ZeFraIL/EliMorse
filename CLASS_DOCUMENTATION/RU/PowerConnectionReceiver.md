# Класс: PowerConnectionReceiver

## 1. Общая информация
*   **Имя класса:** `PowerConnectionReceiver`
*   **Тип:** BroadcastReceiver (Приемник широковещательных сообщений)
*   **Назначение:** Этот класс отслеживает, подключен ли телефон к зарядному устройству. Его главная задача — остановить предупреждения о низком заряде батареи, как только пользователь подключит телефон к сети.
*   **Взаимодействие:** Слушает сигналы системы Android о подключении питания. Может останавливать работу `BatteryLevelReceiver` для экономии ресурсов.

## 2. Переменные (поля класса)
*   **Отсутствуют:** Логика класса строится на методах.

## 3. Методы класса

### Имя метода: `onReceive`
*   **Тип метода:** `public`
*   **Логика работы:**
    1. Проверяет системное намерение (Intent) на предмет текущего статуса зарядки.
    2. Если телефон сейчас заряжается или полностью заряжен, вызывает метод `unregisterBatteryReceiver()`, чтобы прекратить все текущие оповещения о батарее.
*   **Когда вызывается:** Когда пользователь вставляет или вынимает штекер зарядного устройства.

### Имя метода: `showLowBatteryAlert`
*   **Тип метода:** `public static`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `context (Context)` — контекст для показа окна.
*   **Логика работы:** Создает и показывает стандартное диалоговое окно `AlertDialog` с сообщением «зарядите ваш телефон» (charge your phone). Это окно является модальным, то есть пользователь должен нажать «ОК», чтобы закрыть его.
*   **Когда вызывается:** Обычно вызывается классом `BatteryLevelReceiver`, когда уровень заряда становится критически низким.

## 4. Жизненный цикл
*   **Неприменимо:** Компонент живет очень недолго и активируется системными событиями.

## 5. Взаимодействие с интерфейсом (UI)
*   **AlertDialog:** Простое всплывающее окно в системном стиле.

## 6. Взаимодействие с другими компонентами
*   **BatteryLevelReceiver:** Этот класс управляет жизненным циклом монитора батареи, отменяя его регистрацию при восстановлении питания.

## 7. Общая логика класса
`PowerConnectionReceiver` — это **«Выключатель»**. В то время как `BatteryLevelReceiver` — это сама сигнализация, этот класс — тот самый человек, который выключает сирену, как только «чрезвычайная ситуация» (пустая батарея) миновала благодаря подключению зарядки.

## 8. Упрощенное объяснение
Представьте этот класс как **датчик** в гнезде зарядки. Когда вы вставляете кабель, датчик отправляет сообщение: «Теперь телефон в безопасности, можно выключить предупреждающую сирену!».
