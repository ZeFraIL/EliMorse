# Класс: Exercise

## 1. Общая информация
*   **Имя класса:** `Exercise`
*   **Тип:** Activity
*   **Назначение:** Этот экран позволяет пользователю выбрать тип упражнения по азбуке Морзе, которое он хочет выполнить. На выбор предлагается четыре варианта: Числа, Слова, Буквы и Аудирование (на слух).
*   **Взаимодействие:** Наследует `BaseActivity`. В зависимости от выбора пользователя запускает либо `ExerciseGame` (для кодирования текста в Морзе), либо `ListeningExerciseActivity` (для декодирования звука).

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `rbNumbers` | `RadioButton` | Опция упражнения с числами. | Проверяется в `findGameType`. |
| `rbWords` | `RadioButton` | Опция упражнения со словами. | Проверяется в `findGameType`. |
| `rbLetters` | `RadioButton` | Опция упражнения с буквами. | Проверяется в `findGameType`. |
| `rbListening` | `RadioButton` | Опция упражнения на слух. | Проверяется в `findGameType`. |
| `bNext` | `Button` | Кнопка «Далее» для перехода к упражнению. | Запускает процесс перехода. |
| `user` (наследуется) | `User` | Данные текущего пользователя. | Передаются в следующее Activity. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Устанавливает макет `activity_exercise.xml`.
    2. Инициализирует элементы интерфейса через `initElements()`.
    3. Настраивает меню.
    4. Настраивает кнопку «Далее» (`bNext`):
        *   Определяет выбранный тип игры через `findGameType()`.
        *   Если тип выбран, создает `Intent` для перехода к нужному экрану (`ExerciseGame` или `ListeningExerciseActivity`).
        *   Передает тип игры и объект пользователя в следующее Activity.
        *   Если ничего не выбрано, выводит сообщение (Toast) с просьбой сделать выбор.
*   **Когда вызывается:** Когда пользователь нажимает «Упражнения» на главной панели управления.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Получает объект `User` из переданных данных (Intent).
    2. Связывает переменные Java с кнопками и переключателями в XML.
*   **Когда вызывается:** Внутри метода `onCreate`.

### Имя метода: `findGameType`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `int` (константа типа игры)
*   **Логика работы:** Проверяет, какой из RadioButton активен, и возвращает числовую константу из класса `GameConstants`. Если ничего не выбрано, возвращает -1.
*   **Когда вызывается:** При нажатии на кнопку «Далее».

## 4. Жизненный цикл
*   **`onCreate()`**: Подготавливает экран выбора.

## 5. Взаимодействие с интерфейсом (UI)
*   **RadioGroup / RadioButtons:** Используются для выбора одного из взаимоисключающих вариантов упражнения.
*   **Button (`bNext`):** Служит триггером для перехода к экранам практики.

## 6. Взаимодействие с другими компонентами
*   **GameConstants:** Содержит предопределенные числовые значения для каждого типа игры, чтобы избежать ошибок и путаницы между экранами.
*   **Intents:** Используется для передачи ключа `KEY_SENDER` (тип игры) и данных пользователя на следующий экран.

## 7. Общая логика класса
Класс `Exercise` работает как **селектор** (выборщик). Это простое меню, где вы выбираете категорию. Как только вы определились (нажав «Далее»), он передает управление на «тренировочную площадку», сообщая ей, что именно вы хотите тренировать.

## 8. Упрощенное объяснение
Представьте `Exercise` как **автомат с едой**. На автомате есть четыре кнопки (RadioButtons), представляющие разные закуски (Числа, Слова и т.д.). Вы нажимаете одну кнопку, чтобы выбрать, а затем тянете за ручку (кнопка «Далее»). Автомат выдает вам именно ту закуску, которую вы попросили.

---
**Советы по улучшению:**
*   В интерфейсе нет выбора по умолчанию. Было бы лучше, если бы «Буквы» или «Числа» были выбраны сразу, чтобы пользователю не приходилось видеть ошибку, если он просто нажмет «Далее».
*   Метод `findGameType` помечен как `public`, но используется только внутри этого класса; его можно сделать `private`.
