# Класс: HelperDB

## 1. Общая информация
*   **Имя класса:** `HelperDB`
*   **Тип:** Обычный класс (SQLiteOpenHelper)
*   **Назначение:** Этот класс определяет «скелет» базы данных приложения. Он точно указывает, какие таблицы существуют (Пользователи и Упражнения), как называются их столбцы и какие типы данных они хранят. Также он отвечает за создание базы данных при первом запуске и её обновление при изменении структуры.
*   **Взаимодействие:** Используется репозиторием `UserRepository` и активностями, которые сохраняют результаты (например, `ExerciseGame`). Это фундамент для всех постоянных данных в приложении.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `DATABASE_NAME` | `String` | Имя файла («Info.db»). | Используется системой для поиска файла базы. |
| `TABLE_USER` | `String` | Имя таблицы пользователей. | Используется в SQL-запросах. |
| `COLUMN_USER_NAME`| `String` | Столбец для имени пользователя. | Используется в SQL-запросах. |
| `TABLE_EXERCISE`| `String` | Имя таблицы упражнений. | Используется в SQL-запросах. |
| `COLUMN_EXERCISE_MISTAKES`| `String`| Столбец для количества ошибок. | Используется в SQL-запросах. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `public`
*   **Параметры:** `db (SQLiteDatabase)` — объект базы данных.
*   **Логика работы:**
    1. Собирает две длинные строки «CREATE TABLE», используя имена переменных, определенных ранее.
    2. Выполняет эти SQL-команды для физического создания таблиц в памяти телефона.
*   **Когда вызывается:** Автоматически системой Android при самом первом обращении приложения к базе данных.

### Имя метода: `onUpgrade`
*   **Тип метода:** `public`
*   **Логика работы:**
    1. Удаляет старые таблицы.
    2. Вызывает `onCreate()`, чтобы создать новые.
*   **Когда вызывается:** Когда номер версии базы данных (`DATABASE_VERSION`) увеличивается (например, с 1 на 2).
*   **Что важно понимать:** Текущая реализация удаляет *все* данные при обновлении. В реальном приложении следовало бы написать код для переноса данных в новую структуру вместо удаления.

## 4. Жизненный цикл
*   **Неприменимо:** Управляется системой Android по мере необходимости.

## 5. Взаимодействие с другими компонентами
*   **Движок SQLite:** Класс напрямую взаимодействует со встроенным в телефон движком баз данных.
*   **ContentValues:** Используется вместе с этим классом для удобной вставки данных.

## 6. Общая логика класса
`HelperDB` — это **«Архитектор»**. Он рисует план дома (базы данных). Он решает, где будет кухня (Пользователи), а где — гостиная (Упражнения). Сам он в доме не живет, но следит за тем, чтобы дом был построен правильно.

## 7. Упрощенное объяснение
Представьте `HelperDB` как **органайзер**. Он подготавливает большой шкаф с ящиками. Он подписывает ящики: «Пользователи» и «Упражнения». Также он следит за тем, что если вам понадобится шкаф побольше, он выбросит старый и поставит новый, более удобный.

---
**Советы по улучшению:**
*   **Стратегия обновления:** Как уже упоминалось, `DROP TABLE` — это разрушительное действие. Использование `ALTER TABLE` лучше подходит для готовых приложений.
*   **Типы данных:** Убедитесь, что столбец ошибок (`mistakes`) имеет тип `INTEGER` в строке запроса для правильной сортировки и расчетов.
