# 📱 EliMorse - Документация Проекта

## 📌 Описание
EliMorse — это образовательное Android-приложение, предназначенное для изучения и практики азбуки Морзе.
### Основные функции:
- **Перевод:** Преобразование текста в Морзе и обратно в реальном времени.
- **Упражнения:** Интерактивные игры для тренировки цифр, букв и слов.
- **Сигналы:** Воспроизведение Морзе через вспышку фонарика и звук (SoundPool).
- **Система пользователей:** Регистрация и вход для отслеживания прогресса.
- **История:** Просмотр результатов прошлых упражнений из локальной базы данных.
- **Напоминания:** Планируемые уведомления для регулярных тренировок.
- **Доступность:** Голосовая навигация через Text-to-Speech (TTS).

## 🏗️ Архитектура
Проект использует паттерн **MVC (Model-View-Controller)**:
- **Model (Модель):** Классы данных (`User`, `Question`) и доступ к данным (`UserRepository`, `HelperDB`).
- **View (Вид):** XML-файлы разметки.
- **Controller (Контроллер):** Activity (`Translate`, `ExerciseGame`, `LogAndReg`) и фрагменты.

## 📂 Структура проекта
- `java/elia/shapira/elimorse/`: Основные классы логики.
- `res/layout/`: Файлы интерфейса.
- `res/values/strings.xml`: Централизованные текстовые ресурсы.
- `res/raw/`: Аудиофайлы (точки и тире).
- `res/menu/`: Определения всплывающих меню.

## 🧩 Описание классов

### 🔹 MorseTranslator
**Назначение:** Ядро логики перевода.
**Методы:**
- `toMorse(String text)`: Возвращает строку Морзе.
- `toText(String morse)`: Декодирует Морзе (неизвестные коды заменяются на '?').

### 🔹 UserRepository и HelperDB
**Назначение:** Управление локальной базой данных SQLite.
**Методы:**
- `registerUser(User user)`: Сохраняет нового пользователя.
- `findUser(name, password)`: Аутентификация.

### 🔹 TTS_Service
**Назначение:** Фоновое голосовое сопровождение навигации.
**Методы:**
- `speak(String text)`: Озвучивает переходы между экранами.

### 🔹 ExerciseGame
**Назначение:** Игровой цикл тренировок.
**Методы:** `checkAnswer()` (проверка ответа), `saveResults()` (сохранение в БД).

---

## 🔄 Потоки данных
1. Пользователь вводит данные в UI (View).
2. Контроллер (Activity) обрабатывает ввод и обращается к репозиторию.
3. Данные сохраняются или считываются из `HelperDB`.
4. Интерфейс обновляется на основе полученных данных.

## ⚠️ Обработка ошибок
- **Некорректный Морзе:** Заменяется на символ `?`.
- **Пустые поля:** Валидация в `LogAndReg` с выводом Toast.
- **Ошибки парсинга:** Обработка исключений в `Reminder`.
- **Оборудование:** Проверка доступности камеры для фонарика.
