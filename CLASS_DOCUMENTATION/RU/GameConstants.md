# Класс: GameConstants

## 1. Общая информация
*   **Имя класса:** `GameConstants`
*   **Тип:** Обычный класс (Контейнер констант)
*   **Назначение:** Этот класс служит единым источником истины для «Типов игр» в приложении. Он использует целые числа для обозначения различных видов деятельности (Числа, Слова, Буквы, Аудирование).
*   **Взаимодействие:** Используется классами `Exercise`, `ExerciseGame` и `QuestionRepository`, чтобы гарантировать, что все части программы одинаково понимают тип выбранного упражнения.

## 2. Переменные (поля класса)
| Имя | Тип | Значение | Цель |
| :--- | :--- | :--- | :--- |
| `GAME_TYPE_NUMBERS` | `int` | 0 | Идентификатор для упражнения с числами. |
| `GAME_TYPE_WORDS` | `int` | 1 | Идентификатор для упражнения со словами. |
| `GAME_TYPE_LETTERS` | `int` | 2 | Идентификатор для упражнения с буквами. |
| `GAME_TYPE_LISTENING`| `int` | 3 | Идентификатор для упражнения на слух. |

## 3. Методы класса
*   **Отсутствуют:** Этот класс содержит только данные.

## 4. Жизненный цикл
*   **Неприменимо:** Константы доступны всё время, пока приложение находится в памяти.

## 5. Взаимодействие с другими компонентами
*   **Intents:** Эти константы часто передаются как «дополнения» (Extras) в интентах, чтобы сообщить следующему экрану, какой именно режим игры нужно запустить.

## 6. Общая логика класса
`GameConstants` — это **«Свод правил»**. Вместо того чтобы писать «0» в одном файле и «Числа» в другом, все части приложения просто смотрят в этот свод правил и видят, что «0» всегда означает «Числа». Это предотвращает ошибки, когда одна часть приложения думает, что 0 — это одно, а другая часть — что это что-то другое.

## 7. Упрощенное объяснение
Представьте `GameConstants` как **меню с номерами блюд**. Вместо того чтобы кричать повару «Я хочу гамбургер», вы говорите «Мне номер 1». И у официанта, и у повара одно и то же меню, поэтому они оба точно знают, что «номер 1» — это гамбургер.
