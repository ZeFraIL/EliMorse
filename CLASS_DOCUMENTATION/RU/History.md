# Класс: History

## 1. Общая информация
*   **Имя класса:** `History`
*   **Тип:** Activity
*   **Назначение:** Этот экран отображает список всех упражнений, выполненных текущим пользователем. Он показывает дату выполнения, тип упражнения (числа, буквы и т.д.) и количество допущенных ошибок.
*   **Взаимодействие:** Наследует `BaseActivity`. Читает данные из базы данных `HelperDB` (SQLite) и выводит их в виде списка `ListView`.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `alAll` | `ArrayList<Exercises>` | Хранит объекты упражнений, полученные из БД. | Заполняется в `BuildInfo`. |
| `alInfo` | `ArrayList<String>` | Хранит отформатированные строки текста для отображения в списке. | Используется адаптером для вывода на экран. |
| `helperDB` | `HelperDB` | Помощник для работы с БД. | Нужен для открытия соединения с базой данных. |
| `lvInfo` | `ListView` | Элемент интерфейса — список. | Инициализируется в `initElements`. |
| `adapter` | `ArrayAdapter<String>` | Связывает данные из списка `alInfo` с элементом `lvInfo`. | Создается и устанавливается в `BuildInfo`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Логика работы:**
    1. Устанавливает макет `activity_history.xml`.
    2. Вызывает `initElements()` для настройки списка и загрузки данных.
    3. Настраивает клик по меню.
*   **Когда вызывается:** Когда пользователь нажимает «История» (History) на главной панели.

### Имя метода: `BuildInfo`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Открывает базу данных для чтения.
    2. Делает запрос к таблице упражнений.
    3. Проходит по всем строкам результата с помощью «курсора» (`Cursor`).
    4. Для каждой записи проверяет, совпадает ли имя пользователя (`exUser`) с именем текущего вошедшего пользователя.
    5. Если совпадает — создает объект `Exercises` и добавляет красивую строку в список `alInfo`.
    6. Закрывает курсор и базу данных.
    7. Создает адаптер и привязывает его к `ListView`.
*   **Когда вызывается:** Внутри метода `initElements`.
*   **Что важно понимать:** Данные фильтруются вручную в коде (`if(stUser.equals(...))`). Правильнее было бы делать фильтрацию прямо в SQL-запросе к базе данных.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Получает объект текущего пользователя.
    2. Инициализирует списки и помощника БД.
    3. Устанавливает заголовок (например, «История упражнений Элии»).
    4. Вызывает `BuildInfo()` для загрузки данных на экран.
*   **Когда вызывается:** Во время работы `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Отвечает за загрузку и отображение исторических данных при открытии экрана.

## 5. Взаимодействие с интерфейсом (UI)
*   **ListView (`lvInfo`):** Позволяет прокручивать список результатов.
*   **TextView (`tv`):** Отображает текстовый заголовок сверху.

## 6. Взаимодействие с другими компонентами
*   **HelperDB:** Источник всех данных о прошлых тренировках.
*   **Класс Exercises:** Служит моделью данных для удобного хранения информации об одном упражнении.

## 7. Общая логика класса
Класс `History` — это **«Просмотрщик отчетов»**. Он заходит в «архив» приложения (базу данных), перебирает все папки, выбирает только те, что принадлежат текущему пользователю, и аккуратно выписывает их в список на экране, чтобы пользователь мог видеть свой прогресс.

## 8. Упрощенное объяснение
Представьте `History` как **школьный дневник**. Каждый раз, когда вы заканчиваете тренировку (упражнение), приложение записывает «оценку» (количество ошибок) и дату. Когда вы открываете экран истории, это всё равно что открыть дневник, чтобы посмотреть свои прошлые успехи и даты занятий.

---
**Советы по улучшению:**
*   **Производительность:** Сейчас программа загружает *вообще все* упражнения всех пользователей из базы, а потом перебирает их в Java. Если пользователей много, это будет работать медленно. Лучше сразу просить базу данных выдать только нужные строки через условие `WHERE`.
*   **Закрытие ресурсов:** Курсор должен закрываться всегда, даже если данных нет. В текущем коде есть риск оставить его открытым при определенных условиях.
