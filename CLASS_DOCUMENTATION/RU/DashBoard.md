# Класс: DashBoard

## 1. Общая информация
*   **Имя класса:** `DashBoard`
*   **Тип:** Activity
*   **Назначение:** Это главный экран приложения, который появляется после входа в систему. Он выполняет роль «центра управления» или «хаба», где пользователь выбирает, что он хочет делать: тренировать азбуку Морзе, переводить текст, читать справочник, смотреть историю своих успехов или установить напоминание.
*   **Взаимодействие:** Соединяет почти все основные функции приложения. Наследует `BaseActivity`, чтобы использовать общее меню и функции голосовой озвучки (TTS).

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `sharedPreferences`| `SharedPreferences` | Локальное хранилище настроек. | Используется для сохранения и загрузки выбранной пользователем темы (светлая/темная). |
| `user` (наследуется) | `User` | Данные текущего пользователя. | Передается другим экранам при навигации. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Устанавливает макет `activity_dash_board.xml`.
    2. Вызывает `initElements()` для настройки предпочтений и получения данных пользователя.
    3. Настраивает обработчики кликов для всех шести основных кнопок навигации (Упражнения, Перевод, Справочник, Обо мне, История, Напоминание).
    4. Устанавливает слушатель для текста меню, чтобы показывать всплывающее окно.
*   **Когда вызывается:** Когда пользователь успешно вошел в систему и открывается панель управления.

### Имя метода: `showPopupMenu`
*   **Тип метода:** `protected` (переопределенный)
*   **Возвращаемое значение:** `void`
*   **Логика работы:** Похож на метод в `BaseActivity`, но добавляет специфический пункт «Тема» (Theme) для изменения внешнего вида приложения.
*   **Когда вызывается:** При нажатии на текст меню в верхней части экрана.

### Имя метода: `showThemeDialog`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Показывает диалоговое окно с тремя вариантами: Светлая, Темная, Системная тема.
    2. При выборе варианта немедленно меняет тему приложения с помощью `AppCompatDelegate`.
    3. Сохраняет выбор в `SharedPreferences`, чтобы приложение запомнило его при следующем запуске.
*   **Когда вызывается:** При выборе пункта «Тема» во всплывающем меню.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика работы:**
    1. Инициализирует объект `SharedPreferences`.
    2. Извлекает объект `User`, переданный из экрана входа (`LogAndReg`).
*   **Когда вызывается:** Внутри метода `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Подготавливает главный хаб. Это место, где пользователь проводит больше всего времени, переходя между функциями приложения.

## 5. Взаимодействие с интерфейсом (UI)
*   **Кнопки:** `bExercise`, `bTranslate`, `bGuide`, `bAbout`, `bHistory`, `bReminder`. Каждая запускает переход на новый экран.
*   **TextView (`tvMenu`):** Служит якорем для отображения навигационного меню.

## 6. Взаимодействие с другими компонентами
*   **Навигация:** Использует метод `navigateTo()` из `BaseActivity` для переключения на другие экраны с передачей объекта `User`.
*   **Управление темой:** Взаимодействует с системным классом `AppCompatDelegate` для переключения ночного режима.
*   **Сохранение данных:** Использует `SharedPreferences` для сохранения настроек даже после закрытия приложения.

## 7. Общая логика класса
`DashBoard` — это как **«Главное меню»** в видеоигре. Он сам не выполняет сложных вычислений, но предоставляет четкие пути к различным «мирам» (функциям) приложения. Он гарантирует, что данные пользователя (его имя и настройки) передаются на каждый экран, который он посещает.

## 8. Упрощенное объяснение
Представьте `DashBoard` как **«Главный холл»** в отеле. Из этого холла ведут несколько дверей: в спортзал (Упражнения), в библиотеку (Справочник), к администратору (Обо мне) и так далее. В холле также есть выключатель, чтобы включить или выключить свет (Тема). Когда вы выходите из холла в какую-то комнату, администратор (код программы) следит за тем, чтобы ваше имя (объект пользователя) не потерялось.

---
**Советы по улучшению:**
*   Инициализация пользователя в `initElements` должна предусматривать случай, когда данные не были переданы (хотя в обычном режиме это невозможно).
*   Нажатие «Назад» в меню вызывает `finish()`, что может вернуть пользователя на экран входа. Возможно, стоит добавить подтверждение выхода.
