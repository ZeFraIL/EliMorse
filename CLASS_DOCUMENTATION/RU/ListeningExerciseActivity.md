# Класс: ListeningExerciseActivity

## 1. Общая информация
*   **Имя класса:** `ListeningExerciseActivity`
*   **Тип:** Activity
*   **Назначение:** Этот экран позволяет пользователям тренировать навыки декодирования азбуки Морзе на слух. Приложение проигрывает последовательность звуковых сигналов, а пользователь должен напечатать соответствующие буквы или слова.
*   **Взаимодействие:** Наследует `BaseActivity`. Использует `MorsePlayer` для воспроизведения звука, `QuestionRepository` для генерации вопросов и `HelperDB` для сохранения результатов.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `questionIndex` | `int` | Счетчик прогресса (0-9). | Отслеживает, на каком вопросе находится пользователь. |
| `failedAttempts`| `int` | Общее количество ошибок. | Сохраняется в базу данных в конце. |
| `nOfTry` | `int` | Попытки для текущего звукового сигнала. | Используется для показа ответа после 3 неудач. |
| `etAnswer` | `EditText` | Поле для ввода ответа пользователем. | Сравнивается с правильным ответом. |
| `morsePlayer` | `MorsePlayer` | Объект управления звуком. | Проигрывает точки и тире в виде звука. |
| `currentQuestion`| `Question` | Текущий активный вопрос. | Предоставляет строку-ответ. |
| `executorService`| `ExecutorService`| Управление фоновыми потоками. | Используется для сохранения результатов в БД. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Логика работы:**
    1. Устанавливает макет интерфейса.
    2. Инициализирует компоненты через `initElements()`.
    3. Запускает первый вопрос через `loadNextQuestion()`.
    4. Настраивает слушатель для кнопки «Проиграть звук» для запуска `morsePlayer`.
    5. Настраивает кнопку «Отправить» для проверки ответа.
*   **Когда вызывается:** Когда пользователь выбирает «Аудирование» в меню упражнений.

### Имя метода: `loadNextQuestion`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Проверяет, остались ли еще вопросы.
    2. Если да — обновляет интерфейс (текст прогресса, очистка ввода).
    3. Проигрывает звук Морзе для нового вопроса с задержкой в 1 секунду.
    4. Если вопросов больше нет — сохраняет результаты и показывает итоговое диалоговое окно.
*   **Когда вызывается:** При старте и после правильного ответа.

### Имя метода: `checkAnswer`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Сравнивает содержимое `EditText` с текстом вопроса (без учета регистра).
    2. **Если правильно:** увеличивает индекс и загружает следующий вопрос.
    3. **Если неправильно:** увеличивает счетчик ошибок и использованных попыток. После 3 попыток показывает окно неудачи.
*   **Когда вызывается:** При нажатии на кнопку «Отправить».

### Имя метода: `saveResults`
*   **Тип метода:** `private`
*   **Логика работы:** Записывает тип упражнения ("Listening"), количество ошибок и дату в базу данных SQLite.
*   **Когда вызывается:** Когда сессия упражнений закончена.

## 4. Жизненный цикл
*   **`onCreate()`**: Подготавливает сессию.
*   **`onStop()`**: Критически важный метод: вызывает `morsePlayer.stop()`. Это гарантирует, что если пользователь покинет экран, звуковые сигналы немедленно прекратятся.

## 5. Взаимодействие с интерфейсом (UI)
*   **Кнопка (`bPlaySound`):** Повторно воспроизводит текущую последовательность Морзе.
*   **Поле ввода (`etAnswer`):** Где пользователь печатает расшифрованный текст.
*   **Кнопка (`bSubmit`):** Подтверждает ввод ответа.

## 6. Взаимодействие с другими компонентами
*   **MorsePlayer:** Инкапсулирует логику таймингов и использования MediaPlayer.
*   **QuestionRepository:** Предоставляет «секретные» слова для прослушивания.

## 7. Общая логика класса
`ListeningExerciseActivity` следует **слуховому циклу**: Проиграть сигналы -> Пользователь вводит текст -> Проверка логики. Он устраняет разрыв между слуховым восприятием и пониманием текста. Это требует от пользователя переводить в уме код Морзе обратно в слова.

## 8. Упрощенное объяснение
Представьте, что это **цифровой диктант**. Приложение выступает в роли телеграфиста, который отправляет вам сообщение. Ваша задача — внимательно выслушать «писк» и записать то, что говорит телеграфист, в свой блокнот (текстовое поле).
