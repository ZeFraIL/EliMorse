# Класс: BaseActivity

## 1. Общая информация
*   **Имя класса:** `BaseActivity`
*   **Тип:** Activity (абстрактный базовый класс)
*   **Назначение:** Этот класс служит «шаблоном» или «родителем» для других экранов (Activity) приложения. Он содержит общую логику, которая нужна многим экранам, например, навигационное меню и голосовые уведомления (Text-to-Speech). Вместо того чтобы переписывать код меню в каждом экране, другие классы (например, `DashBoard` или `Exercise`) «наследуют» этот класс.
*   **Взаимодействие:** Наследуется большинством других Activity в приложении. Использует `TTS_Service` для озвучивания названий разделов и взаимодействует с различными классами Activity при переходе между ними.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `user` | `User` | Хранит объект текущего вошедшего пользователя. | Используется для отслеживания того, кто использует приложение, на разных экранах. |

## 3. Методы класса

### Имя метода: `showPopupMenu`
*   **Тип метода:** `protected` (защищенный, доступен наследникам)
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `anchorView` | `TextView` | Элемент интерфейса (например, текст «Меню»), к которому будет прикреплено всплывающее окно. |
*   **Логика работы:**
    1. Создает объект `PopupMenu` (всплывающее меню).
    2. Загружает пункты меню из ресурсного файла `total_menu.xml`.
    3. Устанавливает обработчик кликов для пунктов меню (Словарь, Титры, Напоминание, Назад, Выход).
    4. Переходит на соответствующий экран в зависимости от выбранного пункта.
*   **Когда вызывается:** Когда пользователь нажимает на кнопку/текст вызова меню в любом экране, наследующем `BaseActivity`.
*   **Что важно понимать:** Этот метод централизует логику навигации. Если вы захотите добавить новый пункт меню для всех экранов, вам нужно будет изменить его только здесь и в XML-файле.

### Имя метода: `navigateTo`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `activityClass` | `Class<?>` | Класс экрана (Activity), который нужно открыть. |
    | `sectionName` | `String` | Название раздела (для голосового озвучивания). |
*   **Логика работы:**
    1. Создает `Intent` (намерение) для перехода к указанному экрану.
    2. Прикрепляет объект текущего пользователя (`user`) к намерению, чтобы следующий экран «знал», кто вошел в систему.
    3. Вызывает метод `sayWhat()`, чтобы озвучить, куда переходит пользователь.
    4. Запускает новый экран.
*   **Когда вызывается:** Вручную дочерними классами, когда им нужно переключить экран.
*   **Что важно понимать:** Метод автоматизирует передачу данных пользователя и голосовую обратную связь.

### Имя метода: `sayWhat`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `sayThis` | `String` | Текст сообщения или название раздела для озвучивания. |
*   **Логика работы:**
    1. Подготавливает сообщение вида «Вы переходите в раздел [название]».
    2. Запускает службу `TTS_Service` и отправляет ей это сообщение.
*   **Когда вызывается:** Внутри `navigateTo()` или вручную, когда нужно что-то озвучить.
*   **Что важно понимать:** Работа метода зависит от того, правильно ли зарегистрирована служба `TTS_Service` в файле `AndroidManifest.xml`.

## 4. Жизненный цикл
*   **`onCreate()`**: Явно не реализован в `BaseActivity` (предполагается, что дочерние классы вызывают `super.onCreate()`).
*   **`onStart()`, `onResume()` и др.**: Применяется стандартный жизненный цикл Activity, специальная логика в этом базовом классе не добавлена.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** У класса нет своего макета (layout), но он работает с элементами `TextView`, передаваемыми в `showPopupMenu`.
*   **Обработка:** Обрабатывает клики по пунктам меню через `PopupMenu.OnMenuItemClickListener`.

## 6. Взаимодействие с другими компонентами
*   **Intents:** Использует `Intent` для запуска новых Activity и передачи объекта `User`.
*   **Службы (Services):** Запускает `TTS_Service` для обеспечения голосовой связи.

## 7. Общая логика класса
`BaseActivity` — это своего рода «швейцарский нож» для других экранов. Вместо того чтобы каждый экран знал, как показать меню или как «говорить» с пользователем, они просто «просят» своего родителя (`BaseActivity`) сделать это за них.
**Пример использования:**
1. Пользователь находится на главном экране (`DashBoard`) и нажимает «Упражнения».
2. `DashBoard` вызывает `navigateTo(Exercise.class, "Упражнения")`.
3. `BaseActivity` говорит службе TTS произнести: «Вы переходите в раздел Упражнения».
4. `BaseActivity` запускает экран `Exercise` и передает туда данные пользователя.

## 8. Упрощенное объяснение
Представьте, что `BaseActivity` — это **«Свод общих правил»** или **«Главный шаблон»**. Если вы строите разные комнаты в доме (экраны приложения), `BaseActivity` предоставляет стандартные двери и систему внутренней связи. Каждая комната, построенная по этому шаблону, автоматически получает дверь в коридор (меню) и возможность пользоваться динамиками (озвучка текста).
