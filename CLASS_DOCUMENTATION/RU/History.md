# 📱 Документация History (История упражнений)

## 1. Общая информация
*   **Название:** `History`
*   **Тип:** Активность (Activity)
*   **Назначение:** Этот экран отображает список всех упражнений, которые выполнил пользователь. Он показывает дату, тип упражнения и количество допущенных ошибок.
*   **Взаимодействие:** Считывает данные из `HelperDB` и отображает их с помощью `ListView`.

## 2. Переменные (Поля класса)
| Название | Тип | Назначение | Использование |
| :--- | :--- | :--- | :--- |
| `alInfo` | `ArrayList` | Список строк текста, подготовленных для показа. | Каждая строка выглядит так: "Дата - Тип - Ошибки". |
| `lvInfo` | `ListView` | Элемент интерфейса для прокручиваемого списка. | Связан с адаптером. |
| `adapter` | `ArrayAdapter` | "Мост" между данными и интерфейсом. | Берет список `alInfo` и "рисует" его в `ListView`. |

## 3. Методы
### Метод: `BuildInfo`
*   **Тип:** `private`
*   **Что делает:** 
    1. Открывает базу данных для чтения.
    2. Запрашивает все строки из таблицы `Exercises`.
    3. Проходит циклом по результатам (используя `Cursor`).
    4. Проверяет, принадлежит ли запись текущему пользователю.
    5. Форматирует данные в понятную строку и добавляет в список.
    6. Обновляет `ListView`, чтобы пользователь увидел данные.
*   **Вызов:** Автоматически при открытии экрана (в `onCreate`).

## 7. Общая логика
Когда открывается экран, приложение идет к "шкафу с документами" (базе данных), достает все папки с пометкой "Упражнения", отбирает те, что принадлежат вошедшему пользователю, и выводит их списком на экран.

## 8. Объяснение простыми словами
Представьте, что экран `History` — это ваш **личный табель успеваемости**. Каждый раз, когда вы заканчиваете игру, приложение сохраняет вашу "оценку". Этот экран просто открывает журнал оценок и показывает все ваши прошлые результаты, чтобы вы могли видеть свой прогресс.
