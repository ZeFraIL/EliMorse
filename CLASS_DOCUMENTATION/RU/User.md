# Класс: User

## 1. Общая информация
*   **Имя класса:** `User`
*   **Тип:** Обычный класс (модель данных)
*   **Назначение:** Этот класс является «контейнером данных», который представляет человека, использующего приложение. Он хранит всю важную информацию о пользователе: имя, пароль и контактные данные.
*   **Взаимодействие:** Используется почти во всех частях приложения. Хранится в базе данных, передается между экранами через Intent и используется для персонализации интерфейса.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `id` | `int` | Уникальный номер из базы данных. | Позволяет отличать пользователей в БД. |
| `userName` | `String` | Имя пользователя. | Отображается на экранах истории и в главном меню. |
| `userPassword`| `String` | Секретный пароль. | Используется для входа в систему. |
| `userPhone` | `String` | Номер телефона. | Часть профиля пользователя. |
| `userMail` | `String` | Электронная почта. | Часть профиля пользователя. |

## 3. Методы класса

### Имя метода: `User` (Конструктор)
*   **Тип метода:** `public`
*   **Логика работы:** Существует две версии. Первая принимает все поля, включая ID (нужна при чтении из базы). Вторая принимает всё, *кроме* ID (нужна при создании нового пользователя, который еще не сохранен).
*   **Когда вызывается:** При регистрации, входе или загрузке данных из истории.

### Имена методов: `get...` / `set...` (Геттеры и Сеттеры)
*   **Тип метода:** `public`
*   **Логика работы:** Стандартные методы для чтения или обновления закрытых переменных класса.
*   **Когда вызывается:** Каждый раз, когда приложению нужно узнать имя пользователя или обновить его почту.

### Имя метода: `toString`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `String` (строка)
*   **Логика работы:** Создает читаемое описание объекта пользователя для отладки программы.
*   **Когда вызывается:** Обычно для вывода информации в лог разработчика.

## 4. Жизненный цикл
*   **Неприменимо:** Так как это обычный Java-класс, у него нет жизненного цикла Android. Он существует в памяти до тех пор, пока на него есть ссылки.

## 5. Взаимодействие с интерфейсом (UI)
*   **Напрямую нет:** У класса нет визуального интерфейса, но его данные часто отображаются в текстовых полях (`TextView`) по всему приложению.

## 6. Взаимодействие с другими компонентами
*   **Интерфейс Serializable:** Этот класс реализует `Serializable`. Это очень важно в Android, так как позволяет «упаковать» весь объект `User` в Intent и отправить его на другой экран.
*   **UserRepository:** Репозиторий превращает строки из базы данных в объекты `User`.

## 7. Общая логика класса
Класс `User` — это **«Цифровое удостоверение»**. Каждый раз, когда вы переходите на новый экран, вы несете это удостоверение с собой, чтобы приложение знало, кто вы такой.

## 8. Упрощенное объяснение
Представьте класс `User` как **папку с личным делом**. Внутри папки лежат листки бумаги с вашим именем, телефоном и паролем. Когда приложению нужно узнать, кто сейчас тренируется, оно достает эту папку и читает имя на обложке.

---
**Советы по улучшению:**
*   Метод `toString` сейчас включает в себя пароль. Из соображений безопасности лучше исключать секретные данные из строковых представлений, чтобы они случайно не попали в логи.
*   Имена полей немного непоследовательны (где-то `userName`, где-то `userMail`). Приведение к единому стандарту сделает код чище.
