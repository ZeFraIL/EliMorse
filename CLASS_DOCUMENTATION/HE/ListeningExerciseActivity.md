# מחלקה: ListeningExerciseActivity

## 1. מידע כללי
*   **שם המחלקה:** `ListeningExerciseActivity`
*   **סוג:** Activity
*   **מטרת המחלקה:** מסך זה מאפשר למשתמשים לתרגל פענוח של צלילי קוד מורס. האפליקציה משמיעה רצף צפצופים, ועל המשתמש להקליד את האותיות או המילים המתאימות. זהו מבחן ליכולת של המשתמש "לשמוע" ולהבין קוד מורס.
*   **אינטראקציה:** יורשת מ-`BaseActivity`. משתמשת ב-`MorsePlayer` להשמעת השמע, ב-`QuestionRepository` ליצירת שאלות, וב-`HelperDB` לשמירת תוצאות.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `questionIndex` | `int` | מד התקדמות (0-9). | עוקב אחר השאלה בה נמצא המשתמש. |
| `failedAttempts`| `int` | סך כל הטעויות שנעשו. | נשמר לבסיס הנתונים בסיום. |
| `nOfTry` | `int` | ניסיונות עבור הצפצוף הנוכחי. | משמש להצגת התשובה הנכונה לאחר 3 כישלונות. |
| `etAnswer` | `EditText` | קלט טקסט עבור ניחוש המשתמש. | מושווה מול התשובה הנכונה. |
| `morsePlayer` | `MorsePlayer` | אובייקט בקרת שמע. | משמיע את הנקודות והקווים כצליל. |
| `currentQuestion`| `Question` | השאלה הפעילה כרגע. | מספקת את מחרוזת התשובה. |
| `executorService`| `ExecutorService`| מנהל תהליכוני רקע. | משמש לשמירת תוצאות ל-DB. |

## 3. מתודות (שיטות) המחלקה

### שם המתודה: `onCreate`
*   **סוג המתודה:** `protected`
*   **לוגיקה:**
    1. מגדירה את העיצוב.
    2. מאתחלת רכיבים דרך `initElements()`.
    3. מתחילה את השאלה הראשונה דרך `loadNextQuestion()`.
    4. מגדירה מאזין ל-"Play Sound" להפעלת ה-`morsePlayer`.
    5. מגדירה מאזין ל-"Submit" לבדיקת התשובה.
*   **מתי נקראת:** כשהמשתמש בוחר "Listening" בתפריט התרגול.

### שם המתודה: `loadNextQuestion`
*   **סוג המתודה:** `private`
*   **לוגיקה:**
    1. בודקת אם ישנן שאלות נוספות.
    2. אם כן, מעדכנת את הממשק (טקסט התקדמות, ניקוי קלט).
    3. משמיעה את צליל המורס של השאלה החדשה לאחר השהיה של שנייה אחת.
    4. אם אין שאלות נוספות, שומרת תוצאות ומציגה דיאלוג סיום משחק.
*   **מתי נקראת:** בזמן האתחול ולאחר תשובה נכונה.

### שם המתודה: `checkAnswer`
*   **סוג המתודה:** `private`
*   **לוגיקה:**
    1. משווה את תוכן ה-`EditText` ל-`questionText`.
    2. **אם נכון:** מעלה את האינדקס וטוענת את השאלה הבאה.
    3. **אם שגוי:** מעלה את מונה הטעויות ואת הניסיונות שנוצלו. לאחר 3 ניסיונות, מציגה את דיאלוג הכישלון.
*   **מתי נקראת:** כשהמשתמש לוחץ על "Submit".

### שם המתודה: `saveResults`
*   **סוג המתודה:** `private`
*   **לוגיקה:** כותבת את סוג התרגול ("Listening"), מספר הטעויות והתאריך לבסיס הנתונים SQLite.
*   **מתי נקראת:** כשסשן התרגול מסתיים.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`**: מכינה את הסשן.
*   **`onStop()`**: שלב קריטי: קוראת ל-`morsePlayer.stop()`. פעולה זו מבטיחה שאם המשתמש עוזב את המסך, הצפצופים ייפסקו מיד.

## 5. אינטראקציה עם ממשק המשתמש (UI)
*   **כפתור (`bPlaySound`):** משמיע מחדש את רצף המורס הנוכחי.
*   **שדה טקסט (`etAnswer`):** המקום שבו המשתמש מקליד את הטקסט המפוענח.
*   **כפתור (`bSubmit`):** מאשר את הניחוש של המשתמש.

## 6. אינטראקציה עם רכיבים אחרים
*   **MorsePlayer:** עוטף את הלוגיקה של התזמון והשימוש ב-MediaPlayer.
*   **QuestionRepository:** מספק את המילים ה"סודיות" שהמשתמש צריך לשמוע.

## 7. לוגיקה כללית של המחלקה
ה-Activity של ה-`ListeningExercise` עוקב אחר **לולאה שמיעתית**: השמעת צפצופים -> הקלדת המשתמש -> בדיקת לוגיקה. הוא מגשר על הפער בין שמיעה (אודיו) להבנה (טקסט). הוא דורש מהמשתמש לתרגם בראשו ממורס חזרה לאנגלית.

## 8. הסבר במילים פשוטות
חשבו על זה כעל **הכתבה דיגיטלית**. האפליקציה פועלת כמו מפעיל טלגרף ששולח לכם הודעה. התפקיד שלכם הוא להקשיב היטב ל"צפצופים" ולכתוב את מה שהמפעיל אומר במחברת שלכם (שדה הטקסט).

**באגים/שיפורים:**
*   המתודה `saveResults` משתמשת כרגע ב-`user.getUserPassword()` כמזהה המשתמש בעמודת בסיס הנתונים. זהו באג (או לפחות פרקטיקה גרועה); יש להשתמש ב-`user.getUserName()`.
