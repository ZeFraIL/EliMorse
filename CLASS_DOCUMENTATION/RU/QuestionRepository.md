# Класс: QuestionRepository

## 1. Общая информация
*   **Имя класса:** `QuestionRepository`
*   **Тип:** Обычный класс (Фабрика / Репозиторий)
*   **Назначение:** Этот класс является «фабрикой», которая создает контент для упражнений. Он случайным образом выбирает слова, буквы или числа и превращает их в объекты `Question`, используя переводчик для получения правильных ответов.
*   **Взаимодействие:** Используется активностями `ExerciseGame` и `ListeningExerciseActivity`. Полагается на класс `MorseTranslator` для генерации правильных ответов.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `translator` | `MorseTranslator` | Движок кодирования. | Используется для получения Морзе-версии случайных слов. |
| `e_words` | `String[]` | Список из 20 английских слов. | Источник для упражнений со словами. |
| `e_letters` | `String[]` | Список из 26 букв латинского алфавита. | Источник для упражнений с буквами. |

## 3. Методы класса

### Имя метода: `generateQuestion`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `Question` (созданный вопрос)
*   **Параметры:** `gameType (int)` — одна из констант из `GameConstants`.
*   **Логика работы:**
    1. Использует оператор `switch` для проверки нужного типа игры.
    2. **Числа:** генерирует случайное число от 0 до 99.
    3. **Слова:** выбирает случайное слово из массива `e_words`.
    4. **Буквы:** выбирает случайную букву из массива `e_letters`.
    5. Вызывает `translator.toMorse()` для выбранного текста.
    6. Объединяет текст и код Морзе в новый объект `Question`.
*   **Когда вызывается:** Когда начинается упражнение или происходит переход к следующему вопросу.

### Имя метода: `generateQuestions`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `List<Question>` (список вопросов)
*   **Параметры:** `gameType (int)`, `count (int)` — сколько вопросов нужно.
*   **Логика работы:** Простой цикл, который вызывает `generateQuestion` несколько раз для заполнения списка.
*   **Когда вызывается:** Обычно в самом начале упражнения, чтобы подготовить все 10 вопросов сразу.

## 4. Жизненный цикл
*   **Неприменимо:** Служебный класс.

## 5. Взаимодействие с другими компонентами
*   **MorseTranslator:** Необходим для обеспечения правильных ответов.
*   **GameConstants:** Гарантирует, что входной `gameType` соответствует стандартам приложения.

## 6. Общая логика класса
`QuestionRepository` — это **«Генератор вопросов»**. Он похож на человека с мешком слов и словарем. Он достает слово из мешка, находит его перевод в словаре Морзе и выдает вам карточку, на которой есть и то, и другое.

## 7. Упрощенное объяснение
Представьте `QuestionRepository` как **составителя тестов**. Вместо того чтобы человек вручную писал вопросы, этот код «бросает кости», чтобы выбрать случайное слово из своей памяти и подготовить его для вашего экзамена.

---
**Советы по улучшению:**
*   **Дубликаты:** Так как используется класс `Random` без проверки, существует небольшая вероятность получить одно и то же слово дважды за одну сессию.
*   **Логика аудирования:** В методе `generateQuestion` для типа `GAME_TYPE_LISTENING` просто используются слова. Было бы интереснее иметь отдельный список для аудирования или смешивать разные типы данных.
