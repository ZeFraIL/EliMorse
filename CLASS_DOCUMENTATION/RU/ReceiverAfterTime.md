# Класс: ReceiverAfterTime

## 1. Общая информация
*   **Имя класса:** `ReceiverAfterTime`
*   **Тип:** BroadcastReceiver (Приемник широковещательных сообщений)
*   **Назначение:** Этот класс — слушатель «будильника». Когда наступает время, установленное на экране напоминаний (`Reminder`), система Android отправляет сигнал. Этот класс ловит сигнал и выводит уведомление на экран пользователя.
*   **Взаимодействие:** Активируется службой `AlarmManager`. Использует `NotificationManager` для показа уведомления и открывает экран `Welcome`, если пользователь нажмет на уведомление.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `nmc` | `NotificationManagerCompat`| Утилита для уведомлений. | (Определена, но не используется в текущей логике). |

## 3. Методы класса

### Имя метода: `onReceive`
*   **Тип метода:** `public`
*   **Логика работы:**
    1. Показывает небольшое сообщение «Ok» (для отладки).
    2. Вызывает метод `build_and_view_Notification()` для вывода настоящего уведомления.
*   **Когда вызывается:** Автоматически системой Android, когда срабатывает запланированное напоминание.

### Имя метода: `build_and_view_Notification`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Создает `Intent`, указывающий на экран приветствия (`Welcome`).
    2. Упаковывает его в `PendingIntent` (чтобы система могла открыть приложение, даже если оно закрыто).
    3. Собирает уведомление: заголовок («Важно!»), текст сообщения, иконку и цвет.
    4. Проверяет наличие разрешений на отправку уведомлений.
    5. Отправляет готовое уведомление в `NotificationManager` для показа.
*   **Когда вызывается:** Внутри метода `onReceive`.

## 4. Жизненный цикл
*   **Неприменимо:** Работает доли секунды, чтобы показать уведомление, после чего прекращает выполнение.

## 5. Взаимодействие с интерфейсом (UI)
*   **Системное уведомление:** Это единственный «интерфейс», который создает данный класс.

## 6. Взаимодействие с другими компонентами
*   **NotificationChannel:** Использует `CHANNEL_1_ID`, чтобы знать, в какой «канал» отправлять оповещение.
*   **Экран Welcome:** Место, куда попадет пользователь при нажатии на уведомление.

## 7. Общая логика класса
`ReceiverAfterTime` — это **«Посыльный»**. Он тихо сидит в засаде, пока не прозвенит будильник. Затем он вскакивает, пишет записку «Пора потренировать азбуку Морзе!» и приклеивает её в верхнюю часть экрана пользователя.

## 8. Упрощенное объяснение
Представьте этот класс как **секретного агента**. Агент сидит в машине (в фоновом режиме). Ровно в 4 часа дня он получает сигнал по рации. Он тут же подбегает к вашей двери и оставляет записку (уведомление) с надписью: «Пора заниматься!».
