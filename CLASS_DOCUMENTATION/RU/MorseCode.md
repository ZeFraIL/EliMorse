# Класс: MorseCode

## 1. Общая информация
*   **Имя класса:** `MorseCode`
*   **Тип:** Обычный класс (Модель данных)
*   **Назначение:** Этот класс представляет одну запись в словаре азбуки Морзе. Он связывает символ (например, «A») с его представлением на языке Морзе (например, «.-»).
*   **Взаимодействие:** Используется классом `GuideList` для отображения пунктов словаря и может передаваться между экранами через сериализацию.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `mCode` | `String` | Последовательность точек и тире. | Отображается в списках. |
| `mLetter` | `String` | Обычный текстовый символ. | Отображается в списках. |

## 3. Методы класса

### Имя метода: `MorseCode` (Конструктор)
*   **Тип метода:** `public`
*   **Логика работы:** Простое присваивание значений полям.
*   **Когда вызывается:** При чтении файла словаря в `GuideList`.

### Имя метода: `toString`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `String` (строка)
*   **Логика работы:** Возвращает строку в формате «код   буква» для удобного отображения в списке.
*   **Когда вызывается:** При добавлении элементов в `ListView` на экране справочника.

## 4. Жизненный цикл
*   **Неприменимо:** Обычный объект.

## 5. Взаимодействие с другими компонентами
*   **Serializable:** Позволяет отправлять объект через Intent.

## 6. Общая логика класса
`MorseCode` — это **«Этикетка»**. С одной стороны написано «.-», а с другой — «A». Это способ держать эти две части информации неразрывно связанными.

## 7. Упрощенное объяснение
Представьте `MorseCode` как **двустороннюю карточку**. На одной стороне картинка (код Морзе), а на другой — слово (буква). Приложение использует эти карточки, чтобы помочь вам учиться.
