# 🧩 תיעוד HelperDB

## 1. מידע כללי
*   **שם:** `HelperDB`
*   **סוג:** מחלקת Java רגילה (יורשת מ-`SQLiteOpenHelper`)
*   **מטרה:** מחלקה זו אחראית על יצירה וניהול של קובץ מסד הנתונים המקומי בשם `Info.db`. היא מגדירה את המבנה (טבלאות ועמודות) שבו נשמרים כל נתוני המשתמשים ותוצאות התרגילים.
*   **אינטראקציה:** בשימוש על ידי `UserRepository` ואקטיביטיז אחרים שצריכים לשמור תוצאות.

## 2. משתנים (קבועים)
| שם | סוג | מטרה |
| :--- | :--- | :--- |
| `DATABASE_NAME` | `String` | שם הקובץ בטלפון: "Info.db". |
| `TABLE_USER` | `String` | שם טבלת המשתמשים: "Users". |
| `COLUMN_USER_NAME` | `String` | שם העמודה השומרת את שם המשתמש. |
| `TABLE_EXERCISE` | `String` | שם טבלת תוצאות התרגילים: "Exercises". |

## 3. שיטות (Methods)
### שיטה: `onCreate`
*   **מה היא עושה:** השיטה הזו רצה **פעם אחת בלבד** כאשר האפליקציה מותקנת ומסד הנתונים נוצר לראשונה. היא מבצעת פקודות SQL ליצירת הטבלאות `Users` ו-`Exercises`.

### שיטה: `onUpgrade`
*   **מה היא עושה:** רצה אם משנים את גרסת מסד הנתונים (`DATABASE_VERSION`). באפליקציה הזו היא מוחקת את הטבלאות הישנות ויוצרת חדשות.

## 5. מבנה מסד הנתונים
1.  **טבלת משתמשים (Users)**: שומרת ID, שם, סיסמה, אימייל וטלפון.
2.  **טבלת תרגילים (Exercises)**: שומרת ID, סוג משחק, מספר טעויות, תאריך ושיוך למשתמש.

## 7. לוגיקה כללית
המחלקה הזו פועלת כ"אדריכל" של מסד הנתונים. היא בדרך כלל לא מחפשת נתונים; היא רק מוודאת שה"בניין" (מבנה מסד הנתונים) מוכן ויש לו את ה"חדרים" (טבלאות) הנכונים עבור הנתונים.

## 8. הסבר במילים פשוטות
חשבו על `HelperDB` כעל **תוכנית בנייה** לארון תיוק. היא קובעת כמה מגירות יהיו לארון (טבלאות) ואילו תוויות יהיו על כל תיקייה (עמודות). היא מוודאת שכשהאפליקציה מתחילה בפעם הראשונה, הארון נבנה בצורה נכונה כדי שיהיה אפשר לשמור בתוכו מידע.
