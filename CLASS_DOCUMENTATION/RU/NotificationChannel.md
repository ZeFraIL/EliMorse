# Класс: NotificationChannel

## 1. Общая информация
*   **Имя класса:** `NotificationChannel`
*   **Тип:** Application Class (Класс приложения)
*   **Назначение:** Этот класс запускается *раньше* любой Activity при старте приложения. Его задача — создать «канал связи» между приложением и системой уведомлений Android. Начиная с Android 8.0, все уведомления должны принадлежать какому-либо каналу, чтобы пользователь мог ими управлять.
*   **Взаимодействие:** Настраивает системную службу `NotificationManager`. Константа `CHANNEL_1_ID` используется классом `ReceiverAfterTime` для отправки уведомлений.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `CHANNEL_1_ID` | `String` | Уникальный ID для канала напоминаний. | Используется в `NotificationCompat.Builder`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `public`
*   **Логика работы:**
    1. Вызывает родительский метод `super.onCreate()`.
    2. Вызывает `createNotificationChannels()`, чтобы зарегистрировать канал в телефоне.
*   **Когда вызывается:** Автоматически системой Android один раз, как только процесс приложения запускается.

### Имя метода: `createNotificationChannels`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Создает объект канала с высокой важностью (это значит, уведомления могут всплывать сверху и издавать звук).
    2. Устанавливает описание канала (оно будет видно в настройках телефона).
    3. Передает объект канала системному менеджеру `NotificationManager`.
*   **Когда вызывается:** Внутри метода `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Глобальная точка входа для всей логики приложения.

## 5. Взаимодействие с другими компонентами
*   **NotificationManager:** Системная служба, которая регистрирует каналы.
*   **AndroidManifest.xml:** Этот класс должен быть указан в теге `<application android:name=".NotificationChannel" ...>`, иначе он не сработает.

## 6. Общая логика класса
`NotificationChannel` — это **«Регистратура»**. Класс идет в государственное учреждение (систему Android) и регистрирует там номер телефона приложения. Как только номер зарегистрирован, приложению разрешается отправлять «звонки» (уведомления) пользователю.

## 7. Упрощенное объяснение
Представьте этот класс как **установку почтового ящика**. Прежде чем вы сможете получать письма (уведомления), вам нужно пойти на почту и зарегистрировать адрес ящика. Этот класс делает именно это, как только вы «заселяетесь» (запускаете приложение).
