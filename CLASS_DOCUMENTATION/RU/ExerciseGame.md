# Класс: ExerciseGame

## 1. Общая информация
*   **Имя класса:** `ExerciseGame`
*   **Тип:** Activity
*   **Назначение:** Этот экран представляет собой интерактивную среду для тренировки кодирования текста в азбуку Морзе. Пользователю показывают слово, букву или цифру, и он должен ввести правильную последовательность точек и тире. Класс отслеживает ошибки и сохраняет итоговый результат в базу данных.
*   **Взаимодействие:** Наследует `BaseActivity`. Использует `QuestionRepository` для генерации случайных вопросов, `HelperDB` для сохранения результатов и возвращается на `DashBoard` по окончании упражнения.

## 2. Переменные (поля класса)
| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `questionNum` | `int` | Индекс текущего вопроса (от 0 до 9). | Используется для отслеживания прогресса в списке вопросов. |
| `failedAttempts`| `int` | Общее количество ошибок за всю сессию. | Сохраняется в базу данных в конце. |
| `nOfTry` | `int` | Количество попыток для *текущего* вопроса. | Используется для показа диалога неудачи после 3 ошибок. |
| `tvQuestion` | `TextView` | Отображает текст вопроса (например, "CAT"). | Обновляется в `resetAfterQuestion`. |
| `tvInputNumber` | `TextView` | Показывает азбуку Морзе, которую вводит пользователь. | Обновляется в `appendToAnswer`. |
| `stAnswer` | `String` | Хранит введенную пользователем строку Морзе. | Используется для сравнения в `checkAnswer`. |
| `helperDB` | `HelperDB` | Помощник для работы с базой данных. | Используется в `saveResults`. |
| `questions` | `List<Question>`| Список из 10 вопросов для текущей тренировки. | Генерируется в `initElements`. |
| `currentQuestion`| `Question` | Объект текущего вопроса. | Нужен для проверки правильности ответа. |
| `executorService`| `ExecutorService`| Управление фоновым потоком. | Используется для сохранения результатов без «зависания» интерфейса. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Логика работы:**
    1. Устанавливает макет интерфейса.
    2. Вызывает `initElements()` для подготовки вопросов и настройки UI.
    3. Настраивает слушателей для кнопок точки (`.`), тире (`-`) и пробела (`/`).
    4. Настраивает кнопку «Готово» (Ready) для проверки ответа.
    5. Настраивает кнопки «Очистить» (Clear) и «Удалить символ» (Remove).
*   **Когда вызывается:** При запуске Activity.

### Имя метода: `checkAnswer`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Сравнивает `stAnswer` с правильным ответом из `currentQuestion.getMorseAnswer()`.
    2. **Если правильно:** Переходит к следующему вопросу. Если это был последний вопрос — сохраняет результаты и показывает итоговое окно.
    3. **Если неправильно:** Увеличивает счетчик ошибок `failedAttempts` и попыток `nOfTry`. Обновляет индикаторы попыток (O/Ø). После 3 неудач показывает окно с правильным ответом и завершает игру.
*   **Когда вызывается:** При нажатии на кнопку «Готово».

### Имя метода: `appendToAnswer`
*   **Тип метода:** `private`
*   **Параметры:** `s (String)` — символ для добавления.
*   **Логика работы:** Добавляет символ в `stAnswer` и обновляет текст на экране. Ограничение — 30 символов.
*   **Когда вызывается:** При нажатии на кнопки точки, тире или пробела.

### Имя метода: `saveResults`
*   **Тип метода:** `private`
*   **Логика работы:** Использует фоновый поток для вставки данных о типе упражнения, количестве ошибок, дате и пользователе в базу данных через `ContentValues`.
*   **Когда вызывается:** После успешного завершения всех 10 вопросов.

### Имя метода: `initElements`
*   **Тип метода:** `private`
*   **Логика работы:**
    1. Получает пользователя и тип игры из Intent.
    2. Использует `QuestionRepository` для создания 10 случайных вопросов.
    3. Привязывает все UI-компоненты.
    4. Вызывает `resetAfterQuestion()` для запуска первого вопроса.
*   **Когда вызывается:** Внутри `onCreate`.

## 4. Жизненный цикл
*   **`onCreate()`**: Полная настройка игровой сессии.

## 5. Взаимодействие с интерфейсом (UI)
*   **Кнопки ввода:** Точка, Тире, Пробел.
*   **Кнопки управления:** Готово (подтверждение), Очистить все, Удалить один символ (бекспейс).
*   **Индикаторы:** `tvTryNum1`, `tvTryNum2`, `tvTryNum3` меняют вид или цвет, показывая использованные попытки.

## 6. Взаимодействие с другими компонентами
*   **QuestionRepository:** Необходим для получения разнообразного контента.
*   **HelperDB:** Записывает прогресс пользователя для экрана Истории.
*   **AlertDialog:** Используется для уведомлений о завершении игры или проигрыше.

## 7. Общая логика класса
`ExerciseGame` — это **основной движок** для практики. Он работает по циклу: Показать вопрос -> Собрать ввод -> Проверить ответ -> Следующий вопрос. Он ведет учет ваших успехов и гарантирует, что данные будут сохранены навсегда перед тем, как вы покинете экран.

## 8. Упрощенное объяснение
Представьте `ExerciseGame` как **ведущего викторины**. Ведущий показывает вам карточку со словом. У вас есть маленькая клавиатура всего с тремя символами. Вы печатаете те символы, которые считаете правильными. Ведущий следит за вашими очками и ошибками. В конце ведущий записывает ваш результат в большую книгу (базу данных), чтобы вы могли посмотреть его позже.
